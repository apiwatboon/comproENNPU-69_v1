Arithmetic Operators

➗ ตัวดำเนินการคณิตศาสตร์ของภาษาซี

หน้าตาคุ้นเคยทั้งห้าตัว แต่มีพฤติกรรมลับที่ต้องรู้: เครื่องหมาย / ระหว่าง int กับ int จะ "ตัดเศษทิ้ง" อัตโนมัติ ส่วน % เก็บเศษไว้ให้ — สองตัวนี้จึงเป็นคู่หูกันโดยกำเนิด

เครื่องหมายชื่อตัวอย่าง (int)ผลลัพธ์ข้อสังเกต
+บวก17 + 522—
-ลบ17 - 512—
*คูณ17 * 585ใช้ดอกจัน ไม่ใช่ ×
/หาร17 / 53⚠️ int หาร int = int ตัดเศษทิ้ง! (17.0 / 5 จึงจะได้ 3.4)
%มอดุลัส (เศษ)17 % 52ใช้กับจำนวนเต็มเท่านั้น — คู่หูของ /
++ / --เพิ่ม/ลดทีละ 1i++i เพิ่มขึ้น 1เอกลักษณ์ภาษาซี — ย่อมาจาก i = i + 1 จะเจอบ่อยมากในลูป

💡 ภาษาซีไม่มีเครื่องหมายยกกำลัง — ใช้คูณซ้ำ (x * x) หรือฟังก์ชัน pow(x, 2) จากคลัง <math.h>

ตัวอย่างที่ 5.1 — arith.c
int a = 17, b = 5;
printf("หาร (int):  %d\n", a / b);
printf("เศษ:        %d\n", a % b);
printf("หารทศนิยม: %.2f\n", 17.0 / b);
printf("กำลังสอง:  %d\n", b * b);
🤔

% มีไว้ทำไม?

เศษจากการหารคือพระเอกลับของโปรแกรมมิง: n % 2 == 0 ตรวจเลขคู่ · นาที % 60 หาเศษนาทีที่ไม่ครบชั่วโมง · ลำดับ % 7 หาว่าตรงกับวันอะไรของสัปดาห์

👫

/ กับ % เป็นคู่หูกัน (ฉบับ int)

แบ่งของ 17 ชิ้นให้คน 5 คน: แต่ละคนได้ 17 / 5 = 3 ชิ้น (int ตัดเศษให้เอง) เหลือเศษ 17 % 5 = 2 ชิ้น — ใช้คู่กันเมื่อไรที่ต้อง "แบ่งกลุ่มแล้วดูเศษ"

Order of Operations

🥇 ลำดับการคำนวณ — ใครมาก่อนมาหลัง?

ภาษาซีไม่ได้คำนวณซ้ายไปขวาเสมอไป! ลำดับคือ ( ) → * / % → + − (ระดับเดียวกันทำซ้ายไปขวา และตัวเปรียบเทียบมาก่อน && ||) — เลือกนิพจน์แล้วกด "คำนวณทีละขั้น ▶" ดูคอมไพเลอร์ย่อยทีละก้อน

ส่วนที่ สีเหลือง คือก้อนที่จะถูกคำนวณในขั้นถัดไป (ตัวแปรทุกตัวเป็น int)
💣

กับดักระดับตำนานของภาษาซี: 5 / 9 = 0

สูตรแปลงอุณหภูมิ c = 5 / 9 * (f - 32); ให้คำตอบ 0 เสมอ — เพราะ 5 / 9 เป็น int หาร int ได้ 0 ก่อนจะคูณอะไรทั้งนั้น! ต้องเขียน c = 5.0 / 9.0 * (f - 32); ให้เป็นทศนิยมตั้งแต่ต้น — บั๊กตัวนี้อยู่ในข้อสอบและชีวิตจริงมาห้าสิบปีแล้ว อย่าเป็นเหยื่อรายต่อไป

ตัวอย่างที่ 5.2 — cast.c (วิธีปราบการหารแบบ int)
int sum = 17, n = 4;
printf("เฉลี่ยแบบ int:    %d\n", sum / n);
printf("เฉลี่ยที่ถูกต้อง: %.2f\n", (float) sum / n);   // (float) = แปลงชนิดชั่วคราว
User Input with scanf

⌨️ scanf — รับข้อมูลจากผู้ใช้ และเครื่องหมาย & ที่ห้ามลืม

คู่แฝดของ printf: scanf("ตัวกำหนดรูปแบบ", &ตัวแปร); หยุดรอให้ผู้ใช้พิมพ์แล้วกด Enter — ใช้ %d %f %c ชุดเดียวกับ printf แต่มีของเพิ่มหนึ่งชิ้น: เครื่องหมาย & หน้าชื่อตัวแปร (บอก "ตำแหน่งกล่อง" ให้ scanf เอาค่าไปวาง)

ตัวอย่างที่ 5.3 — input_age.c (สมมุติผู้ใช้พิมพ์ 19)
int age;
printf("อายุของคุณ: ");
scanf("%d", &age);   // อย่าลืม & !
printf("ปีหน้าคุณจะอายุ %d ปี\n", age + 1);
ตัวอย่างที่ 5.4 — รับหลายค่า + ทศนิยม
int w, h;
float price;
scanf("%d %d", &w, &h);   // รับ 2 ค่าในบรรทัดเดียว
scanf("%f", &price);      // float ใช้ %f (double ใช้ %lf)
printf("พื้นที่ %d ตร.ม. ราคา %.2f บาท\n", w * h, price);
☠️

ลืม & แล้วเป็นอะไร?

scanf("%d", age); (ไม่มี &) — คอมไพล์ "ผ่าน" แต่ตอนรันโปรแกรมพังหรือค่าหายเงียบ ๆ เพราะ scanf ไม่รู้ตำแหน่งกล่องจริงของตัวแปร — นี่คือบั๊กยอดฮิตอันดับหนึ่งของบทนี้ จำท่าให้ขึ้นใจ: printf ไม่ใส่ & · scanf ใส่ & เสมอ (ยกเว้นสตริง ซึ่งจะเจอบทที่ 11)

Comparison & Logical Operators

⚖️ เปรียบเทียบและตรรกะ — ภาษาของการตัดสินใจ

ผลของการเปรียบเทียบในภาษาซีคือ int: 1 = จริง, 0 = เท็จ (ไม่มี True/False!) — และนี่คือสิ่งที่จะอยู่ใน "ข้าวหลามตัด" ของ if สัปดาห์หน้า ระวัง: เท่ากับใช้ == สองตัว เพราะ = ตัวเดียวคือกำหนดค่า

เครื่องหมายความหมายตัวอย่าง (a=7, b=3)ผล
==เท่ากับa == b0 (เท็จ)
!=ไม่เท่ากับa != b1 (จริง)
>มากกว่าa > b1 (จริง)
<=น้อยกว่าหรือเท่ากับa <= b0 (เท็จ)
&&และ — จริงทั้งคู่จึงจริงa > 0 && b > 90 (เท็จ)
||หรือ — จริงสักตัวก็จริงa > 0 || b > 91 (จริง)
!นิเสธ — กลับค่า!(a > b)0 (เท็จ)

Putting It Together

🛠️ ใช้งานจริง: เครื่องแปลงวินาที → ชั่วโมง/นาที/วินาที

โจทย์คลาสสิกที่ / กับ % ของจำนวนเต็มเปล่งประกาย: เซ็นเซอร์จับเวลาเครื่องจักรรายงานมาเป็น "วินาทีรวม" แต่หัวหน้าต้องการ ชม./นาที/วิ — การหารแบบตัดเศษของ int ทำหน้าที่ได้พอดีเป๊ะ

ตัวอย่างที่ 5.5 — time_convert.c
int total, hours, remain, minutes, seconds;
scanf("%d", &total);
hours   = total / 3600;   // int หาร int ตัดเศษให้เอง
remain  = total % 3600;
minutes = remain / 60;
seconds = remain % 60;
printf("%d ชม. %d นาที %d วิ\n", hours, minutes, seconds);

Check Your Understanding

📝 แบบทดสอบท้ายบท (10 ข้อ)

สามจุดตาย: int หาร int ตัดเศษ · scanf ต้องมี & · ผลเปรียบเทียบคือ 1/0

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

หัวใจของบทนี้คือ "กับดักการหารจำนวนเต็ม" และการใช้ scanf ให้ถูก — เข้าใจ 5 ข้อนี้แล้วทำแบบทดสอบได้สบาย

1

กับดัก: int / int = int

จำนวนเต็มหารจำนวนเต็ม ผลลัพธ์ถูก ตัดทศนิยมทิ้ง เสมอ

int a = 7, b = 2;
printf("%d", a / b);   // 3 ไม่ใช่ 3.5

float c = 5.0 / 9;     // ต้องมี .0 อย่างน้อยตัวหนึ่ง

2

ตัวดำเนินการมอดูลัส %

% = เศษจากการหาร ใช้เช็คเลขคู่/คี่ และแยกหลัก

17 / 5 = 3      17 % 5 = 2
if (n % 2 == 0)   // n เป็นเลขคู่

3

ลำดับความสำคัญ

* / % ทำก่อน + - เสมอ

2 + 3 * 4 = 2 + 12 = 14   (ไม่ใช่ 20)

4

scanf ต้องมีเครื่องหมาย &

รับค่าเข้าตัวแปรต้องใส่ & หน้าชื่อตัวแปร (ต่างจาก printf ที่ใส่ชื่อตรง ๆ) — ลืม & โปรแกรมพังตอนรัน

int age;
scanf("%d", &age);   // อย่าลืม &

5

ตัวดำเนินการเปรียบเทียบ/ตรรกะ

นิพจน์เปรียบเทียบให้ค่า 1 (จริง) หรือ 0 (เท็จ) · && = และ (ต้องจริงทั้งคู่)

8 > 3            // มีค่าเป็น 1
score>=50 && score>=80   // 65 → เท็จ (0)

💡

จุดที่ข้อสอบชอบถาม

โจทย์คำนวณที่ผลออกมา 0.000000 เกือบทุกครั้งคือกับดัก int division — มองหาว่ามีตัวเลขทศนิยม (.0) หรือยัง
