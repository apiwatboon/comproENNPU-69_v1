The for Loop

🔁 for — วนซ้ำเมื่อรู้จำนวนรอบ

หัวลูป for ของภาษาซีมี 3 ช่องคั่นด้วย ; : for (เริ่มต้น; เงื่อนไขวนต่อ; ปรับค่า) — เช่น for (i = 1; i <= 5; i++) อ่านว่า "เริ่มที่ 1, วนตราบใดที่ i ≤ 5, จบรอบแล้วเพิ่ม i ทีละ 1"

1️⃣

เริ่มต้น (ทำครั้งเดียว)

i = 1 — ตั้งค่าตัวนับก่อนเข้าลูป ทำแค่ครั้งแรกครั้งเดียวเท่านั้น

2️⃣

เงื่อนไข (ตรวจทุกรอบ)

i <= 5 — ตรวจ "ก่อนทำ" ทุกรอบ จริงค่อยทำบล็อก เท็จเมื่อไรออกจากลูปทันที

3️⃣

ปรับค่า (ท้ายทุกรอบ)

i++ — ทำหลังจบบล็อกแต่ละรอบ (i++ คือ i = i + 1 — เอกลักษณ์ภาษาซีจากคาบที่ 5!)

ตัวอย่างที่ 7.1 — multiplication.c แม่สูตรคูณใน 3 บรรทัด
for (int i = 1; i <= 5; i++) {
    printf("3 x %d = %d\n", i, 3 * i);
}

3 บรรทัดได้ผลลัพธ์ 5 บรรทัด — เปลี่ยนเป็น i <= 1000000 ก็ได้ผลลัพธ์ล้านบรรทัด โดยโค้ดยาวเท่าเดิม นี่แหละพลังของลูป (ก้าวทีละ 2 ก็ได้: i += 2)

The Accumulator Pattern

🥤 ท่าไม้ตายที่ต้องจำ: ตัวสะสม (Accumulator)

"หาผลรวม 1 ถึง 5" คือโจทย์คลาสสิกของรูปแบบที่ใช้บ่อยที่สุดในโลกของลูป: ตั้งตัวสะสมเป็น 0 → วนเติมค่าทีละรอบ → ได้ผลรวมตอนจบ กด "รันทีละขั้น ▶" แล้วจับตาดูหัวลูปบรรทัด 2 ที่ถูกแวะซ้ำทุกรอบ

sum_loop.c
1int total = 0;
2for (int i = 1; i <= 5; i++) {
3 total = total + i;
4}
5printf("%d", total);

🎛️ แผงควบคุม

ขั้นที่บรรทัดitotal
— กดรันทีละขั้นเพื่อเริ่ม —
— หน้าจอผลลัพธ์ (Console) —
สังเกตว่าบรรทัด 2 จะถูกแวะซ้ำทุกรอบ — มันคือ "ด่านเช็ค" เงื่อนไข i <= 5 และจุดปรับค่า i++
ตัวอย่างที่ 7.2 — even_sum.c (ปิดตำนานงานคาบที่ 3 ในสี่บรรทัด)
int sum = 0;
for (int i = 2; i <= 20; i += 2) {
    sum = sum + i;            // ตัวสะสม + ก้าวทีละ 2
}
printf("ผลรวมเลขคู่ 2-20 = %d\n", sum);
while & do-while

⏳ while และ do-while — วนซ้ำเมื่อรู้แค่ "เงื่อนไข"

บางงานไม่รู้ล่วงหน้าว่ากี่รอบ: ถามรหัสจนกว่าจะถูก รอจนเซ็นเซอร์ตอบกลับ — while (เงื่อนไข) { } ตรวจก่อนทำ ส่วนน้องของมัน do { } while (เงื่อนไข); ทำก่อนหนึ่งรอบแล้วค่อยตรวจ

ตัวอย่างที่ 7.3 — countdown.c
int count = 3;
while (count > 0) {
    printf("%d\n", count);
    count--;          // สำคัญมาก!
}
printf("ปล่อยจรวด!\n");
ตัวอย่างที่ 7.4 — pin.c (do-while: ถามจนกว่าจะถูก)
int pin;
do {
    printf("ใส่รหัส PIN: ");
    scanf("%d", &pin);
} while (pin != 1234);   // ผิดก็วนถามใหม่
printf("ยินดีต้อนรับ!\n");
🧭

เลือกอะไรดี?

รู้จำนวนรอบ → for (สูตรคูณ 12 แม่)
รู้แค่เงื่อนไขหยุด → while (วนจนค่าคลาดเคลื่อน < 0.001)
ต้องทำอย่างน้อย 1 รอบ → do-while (เมนู, ถามรหัส — ต้องถามก่อนถึงรู้ว่าผิด)

☠️

ลูปไม่รู้จบ (Infinite Loop)

ลบบรรทัด count--; ออก — count ค้างที่ 3 ตลอดกาล เงื่อนไขจริงชั่วนิรันดร์ โปรแกรมค้าง! จำคาบที่ 3: อัลกอริทึมต้องมี Finiteness ทุกลูปต้องมีบรรทัดขยับเข้าจุดจบ (ค้างแล้วกด Ctrl + C บังคับหยุด)

🛑

break — ประตูฉุกเฉิน

คำสั่ง break; ออกจากลูปทันทีไม่ว่าเงื่อนไขจะยังจริงอยู่ — ตัวเดียวกับที่ใช้ใน switch! ใช้คู่กับ if ในลูป เช่น เจอสินค้าที่ค้นหาแล้วก็เลิกหาต่อ

ตัวอย่างที่ 7.5 — find_break.c (เจอแล้วหยุดทันที ไม่หาต่อให้เสียเวลา)
// หาเลขตัวแรกในช่วง 50-60 ที่หาร 7 ลงตัว
for (int i = 50; i <= 60; i++) {
    if (i % 7 == 0) {
        printf("เจอแล้ว! %d หาร 7 ลงตัว\n", i);
        break;                // ออกจากลูปทันที
    }
    printf("%d ไม่ใช่...\n", i);
}
Nested Loops

🪜 ลูปซ้อนลูป: เครื่องวาดลวดลายดาว

ลูปนอกคุม "แถว" ลูปในคุม "คอลัมน์" — ลูปนอก 1 รอบ ลูปในวนจนจบชุด รอบรวม = นอก × ใน — เลือกลวดลาย ปรับจำนวนแถว แล้วกดวาด ดูว่าลูปซ้อน 4 บรรทัดเสกอะไรได้บ้าง

— เลือกลวดลายแล้วกดวาด —
pattern.c
int n = 5;
for (int i = 1; i <= n; i++) {
    for (int j = 1; j <= i; j++)
        printf("*");
    printf("\n");
}

เคล็ดลับอ่านลูปซ้อน: printf("*") ในลูปในพิมพ์ "ต่อบรรทัดเดิม" ส่วน printf("\n") หลังจบลูปในคือการ "ขึ้นแถวใหม่" — i คุมว่าแถวนี้พิมพ์กี่ดวง j เป็นคนนับ

Putting It Together

🌡️ ใช้งานจริง: สแกนหาอุณหภูมิสูงสุดของเครื่องจักร

เซ็นเซอร์วัดอุณหภูมิเครื่องจักรทุกชั่วโมง 7 ค่า — หาค่าสูงสุดโดย "จำแชมป์ปัจจุบันไว้ แล้ววนเทียบทีละตัว ใครชนะก็ขึ้นเป็นแชมป์แทน" กด "เทียบตัวถัดไป ▶" ดูลูปสแกนทีละแท่ง

max_t = —  |  กำลังเทียบชั่วโมงที่ —
โค้ดเบื้องหลัง: max_t = temps[0]; แล้ววน for เทียบ — if (temps[i] > max_t) ก็อัปเดตแชมป์ (สีชมพู = แชมป์ปัจจุบัน, สีเหลือง = ตัวที่กำลังเทียบ)

📝 แบบทดสอบท้ายคาบ (10 ข้อ)

อ่านหัวลูปให้ครบสามช่องก่อนตอบเสมอ: เริ่มที่ไหน วนถึงเมื่อไร ก้าวทีละเท่าไร

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

ฝึก "นับรอบลูป" และ "ไล่ค่าตัวสะสม" ให้คล่อง พร้อมแยกแยะ for / while / do-while และระวังลูปไม่รู้จบ

1

ลูป for: นับรอบ

โครงสร้าง for(เริ่ม; เงื่อนไข; เปลี่ยนค่า) เหมาะเมื่อ "รู้จำนวนรอบ"

for (int i = 0; i < 5; i++)
// ทำ 5 รอบ: i = 0,1,2,3,4

for (int i = 2; i <= 8; i += 2)
// พิมพ์ 2 4 6 8

2

ตัวสะสม (accumulator)

ตั้งค่าเริ่มต้นก่อนลูป แล้วบวกเพิ่มในลูป

int total = 0;
for (int i = 1; i <= 3; i++)
    total = total + i;  // 1+2+3 = 6

3

while vs do-while

while เช็คเงื่อนไขก่อน (อาจไม่ทำเลย) เหมาะเมื่อไม่รู้จำนวนรอบ · do-while ทำก่อน 1 รอบแล้วค่อยเช็ค (รับประกันทำอย่างน้อย 1 ครั้ง เช่นเมนู)

4

กับดักลูปไม่รู้จบ

ถ้าไม่มีบรรทัด "แก้ค่า" ตัวแปรเงื่อนไข ลูปจะวนไม่มีวันจบ

int x = 3;
while (x > 0) { printf("%d", x); }
// ลืม x-- → วนตลอด

5

ลูปซ้อน (nested loop)

จำนวนครั้งทั้งหมด = รอบนอก × รอบใน

for(i=0;i<3;i++)
  for(j=0;j<4;j++) printf("ตรวจ");
// 3 × 4 = 12 ครั้ง

💡

จุดที่ข้อสอบชอบถาม

โจทย์ไล่โค้ดหาค่าตัวสะสม/นับรอบ และ "หาค่าสูงสุด" ที่ต้องตั้ง max = arr[0] ก่อนลูป (กันค่าเริ่มต้นผิด) — ฝึก trace ทีละรอบ
