Learning Objectives

วัตถุประสงค์การเรียนรู้

อธิบายเหตุผลที่ต้องเก็บข้อมูลเป็นไฟล์ และความต่างของโหมด "r" / "w" / "a" ได้
เขียนข้อมูลลงไฟล์ด้วย fopen + fprintf และปิดด้วย fclose ได้ครบวงจร
อ่านไฟล์ทีละบรรทัดด้วย fgets และตรวจสอบการเปิดไฟล์ล้มเหลว (NULL) ได้
อธิบายโครงสร้างไฟล์ CSV และแยกคอลัมน์ด้วย sscanf ได้
เขียนโปรแกรมอ่าน CSV มาสรุปเป็นรายงาน (จำนวน ผลรวม เฉลี่ย สูงสุด-ต่ำสุด) ได้
Writing Files

💾 เขียนไฟล์ — ให้ข้อมูลอยู่รอดหลังปิดโปรแกรม

สามขั้นตอนตายตัว: fopen เปิด → fprintf เขียน → fclose ปิด — fopen คืน "ที่จับไฟล์" ชนิด FILE * ไว้ใช้อ้างถึงไฟล์นั้น และ fprintf ก็คือ printf ที่เขียนลงไฟล์แทนหน้าจอ (รูปแบบ % เหมือนกันเป๊ะ)

ตัวอย่างที่ 12.1 — save_log.c
FILE *fp = fopen("log.txt", "w");
if (fp == NULL) {            // เปิดไม่สำเร็จ?
    printf("เปิดไฟล์ไม่ได้\n");
    return 1;
}
fprintf(fp, "เริ่มกะเช้า 08:00\n");
fprintf(fp, "เครื่องที่ %d หยุดซ่อม\n", 3);
fclose(fp);                  // ปิดเสมอ!
printf("บันทึกแล้ว\n");

🚦 สามโหมดที่ต้องแยกให้ออก

โหมดชื่อพฤติกรรม
"r"readอ่านอย่างเดียว — ไฟล์ไม่มีอยู่ fopen คืน NULL
"w"writeเขียนใหม่ — ⚠️ ของเดิมถูกลบเกลี้ยงทันทีที่เปิด!
"a"appendเขียนต่อท้าย — ของเดิมอยู่ครบ เหมาะกับ log

⚠️ จำให้ขึ้นใจ: เปิดโหมด "w" กับไฟล์รายงานทั้งเดือน = ข้อมูลหายหมดก่อนเขียนตัวแรกด้วยซ้ำ — งาน log รายวันให้ใช้ "a"

🔒 และต่างจากบางภาษา — ภาษาซีไม่ปิดไฟล์ให้อัตโนมัติ: ลืม fclose ข้อมูลอาจเขียนไม่ครบเพราะค้างอยู่ในบัฟเฟอร์

ตัวอย่างที่ 12.2 — append.c (พิสูจน์โหมด "a": เขียนสองรอบ ของเดิมไม่หาย)
FILE *fp = fopen("shift.log", "a");
fprintf(fp, "กะเช้า: ผลิตได้ 120 ชิ้น\n");
fclose(fp);

fp = fopen("shift.log", "a");   // เปิดซ้ำด้วย "a" — ต่อท้าย ไม่ทับ
fprintf(fp, "กะบ่าย: ผลิตได้ 98 ชิ้น\n");
fclose(fp);

char line[100];
fp = fopen("shift.log", "r");    // อ่านกลับมาพิสูจน์
while (fgets(line, 100, fp) != NULL) {
    printf("%s", line);
}
fclose(fp);
Reading Files

📖 อ่านไฟล์ — fgets ทีละบรรทัด ในลูป while ที่คุ้นเคย

fgets(line, ขนาด, fp) อ่านหนึ่งบรรทัดใส่สตริง และคืน NULL เมื่อหมดไฟล์ — จึงเข้าคู่กับ while ได้พอดี จุดเดียวที่ต้องระวัง: บรรทัดที่อ่านมามีอักขระ \n ติดท้ายมาด้วย

ตัวอย่างที่ 12.3 — read_log.c อ่าน log.txt ที่เพิ่งบันทึก (ผลบันทึกไว้ เพราะไฟล์อยู่ในเครื่องคุณ)
FILE *fp = fopen("log.txt", "r");
char line[100];
int count = 0;
while (fgets(line, 100, fp) != NULL) {
    count++;
    printf("บรรทัด %d: %s", count, line);  // line มี \n อยู่แล้ว
}
fclose(fp);
printf("รวม %d บรรทัด\n", count);
🔁

ความรู้เก่าใช้ได้หมด

while วนจน NULL (บท 7) · line คือ char array (บท 11) · ตัวสะสมนับบรรทัด (บท 7) — คาบนี้ไม่มีท่าใหม่ มีแต่สนามใหม่

👻

ระวังตัว \n ล่องหน

fgets เก็บ \n ท้ายบรรทัดมาด้วย — เทียบ strcmp(line, "END") จะไม่มีวันเท่า เพราะจริง ๆ คือ "END\n" ต้องตัด \n ทิ้งก่อน — บั๊กล่องหนตัวจริงของคาบนี้

🛡️

ตรวจ NULL ทุกครั้งที่ fopen

เปิดโหมด "r" กับไฟล์ที่ไม่มี → fopen คืน NULL — ใช้ fp ต่อโดยไม่ตรวจ โปรแกรมพังทันที (crash) จึงต้อง if (fp == NULL) เสมอ เป็นมารยาทพื้นฐานของโปรแกรมไฟล์

CSV Format

📊 CSV — ภาษากลางของข้อมูลทุกวงการ

CSV (Comma-Separated Values) คือไฟล์ข้อความที่คั่นคอลัมน์ด้วยจุลภาค — Excel, เครื่องจักร CNC, ระบบคลังสินค้า, เซ็นเซอร์ IoT ส่งออกเป็น CSV ได้ทั้งนั้น กุญแจไขมันในภาษาซีคือ sscanf: scanf ที่อ่านจาก "สตริง" แทนคีย์บอร์ด

machines.csv — หน้าตาไฟล์จริง
machine,hours,temp
PUMP-01,8,72.5
MIXER-01,9,84.2
CONV-01,7,61.8
machinehourstemp
PUMP-01872.5
MIXER-01984.2
CONV-01761.8

ไฟล์เดียวกัน มองสองมุม: ข้อความดิบ vs ตาราง — บรรทัดแรกคือ "หัวตาราง" (header) ต้องข้ามตอนคำนวณ!

ตัวอย่างที่ 12.4 — parse_csv.c (แยกหนึ่งบรรทัดเป็นข้อมูล)
char line[100] = "PUMP-01,8,72.5";
char name[30];
int hours;
float temp;

// %[^,] = "อ่านไปจนกว่าจะเจอจุลภาค"
sscanf(line, "%29[^,],%d,%f", name, &hours, &temp);

printf("%s ทำงาน %d ชม. อุณหภูมิ %.1f°C\n",
       name, hours, temp);
🧩

sscanf คือเพื่อนเก่าในชุดใหม่

scanf อ่านจากคีย์บอร์ด · fscanf อ่านจากไฟล์ · sscanf อ่านจากสตริง — สามพี่น้องใช้ตัวกำหนดรูปแบบชุดเดียวกันหมด ท่ามาตรฐานอ่าน CSV จึงเป็น: fgets อ่านทั้งบรรทัดก่อน แล้ว sscanf ค่อยแกะคอลัมน์ — แยกหน้าที่ชัดเจน ตรวจบรรทัดเสียได้ง่าย (sscanf คืนค่า "จำนวนช่องที่แกะสำเร็จ" — แกะได้ไม่ครบ 3 = บรรทัดเสีย!)

ตัวอย่างที่ 12.5 — report_write.c (ครบวงจร: คำนวณ → เขียนรายงานลงไฟล์)
int qty[4] = {12, 7, 25, 9};
int sum = 0;
for (int i = 0; i < 4; i++) sum = sum + qty[i];

FILE *fp = fopen("report.txt", "w");
fprintf(fp, "สินค้ารวม %d ชิ้น เฉลี่ยลังละ %.2f\n", sum, (float) sum / 4);
fclose(fp);
printf("เขียน report.txt เรียบร้อย (รวม %d ชิ้น)\n", sum);

ข้อมูลซ้าย โปรแกรมตรงกลาง รายงานขวา — แก้ตัวเลขในไฟล์ได้เลย (ลองเปลี่ยนอุณหภูมิ เพิ่มเครื่องใหม่ หรือพิมพ์บรรทัดเสีย ๆ ดูว่าโปรแกรมรับมืออย่างไร) แล้วกดประมวลผล

sensors.csv — แก้ไขได้

Check Your Understanding

📝 แบบทดสอบท้ายคาบ (10 ข้อ)

สามจุดตาย: โหมด "w" ลบของเดิม · fopen ต้องตรวจ NULL · fgets มี \n ติดมา

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

เน้น "โหมดเปิดไฟล์" และการอ่าน CSV ให้ปลอดภัย — จำ w/a/r และอย่าลืมเช็ค NULL ทุกครั้ง

1

ทำไมต้องบันทึกลงไฟล์

ตัวแปรอยู่ใน RAM หายเมื่อปิดโปรแกรม · ไฟล์เก็บข้อมูลถาวร เปิดมาใช้ต่อวันหลังได้

2

โหมดเปิดไฟล์ w / a / r

"w" เขียนใหม่ — ลบเนื้อหาเดิมทิ้งทั้งหมด · "a" เขียนต่อท้าย (เหมาะกับ log ที่รันซ้ำ) · "r" อ่านอย่างเดียว

FILE *fp = fopen("log.txt", "a");  // ต่อท้าย ไม่ลบของเดิม

3

เช็ค fp == NULL เสมอ

ถ้าเปิดไฟล์ไม่สำเร็จ (ไม่มีไฟล์/ไม่มีสิทธิ์) fopen คืน NULL ถ้าไม่เช็คแล้วใช้ต่อ โปรแกรมพังทันที

if (fp == NULL) { printf("เปิดไฟล์ไม่ได้"); return 1; }

4

อ่านบรรทัดด้วย fgets (ระวัง \n)

fgets อ่านมาทั้ง \n ติดท้ายด้วย จึงทำให้ strcmp(line,"END") ไม่เท่ากับ 0 (เพราะจริง ๆ คือ "END\n")

5

CSV + sscanf

CSV = Comma-Separated Values คั่นคอลัมน์ด้วยจุลภาค , · sscanf แกะข้อมูลจากสตริง (ต่างจาก scanf ที่รับจากแป้นพิมพ์) · %[^,] = อ่านจนกว่าจะเจอ ,

sscanf(line, "%29[^,],%d,%f", name, &qty, &price);

💡

จุดที่ข้อสอบชอบถาม

จำ "w" ลบของเดิม ส่วน log ใช้ "a" · ไฟล์ CSV ที่มีหัวตาราง 1 บรรทัด ข้อมูลจริง = จำนวนบรรทัด − 1 · เช็ค NULL ทุกครั้ง
