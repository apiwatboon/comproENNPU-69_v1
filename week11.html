Strings as char Arrays

🔤 สตริง = ขบวนรถไฟของ char ที่มีโบกี้ปิดท้าย

ประกาศ: char code[20] = "ENGINE"; — คอมไพเลอร์แอบเติมอักขระพิเศษ \0 (null) ปิดท้าย ให้เสมอ เพื่อบอกทุกฟังก์ชันว่า "ข้อความจบตรงนี้" — จองที่จึงต้องเผื่อ \0 หนึ่งช่องเสมอ

ตัวอย่างที่ 11.1 — string_basics.c
#include <string.h>   // คลังฟังก์ชันสตริง

char code[20] = "ENGINE";
printf("%s\n", code);        // ทั้งข้อความใช้ %s
printf("%c\n", code[0]);     // ตัวเดียวใช้ %c
printf("%d\n", strlen(code)); // ความยาว (ไม่นับ \0)
code[0] = 'A';                // แก้ทีละตัวได้!
printf("%s\n", code);

⚠️ สองกับดักประจำบท

1️⃣ กำหนดค่าใหม่ทั้งก้อนด้วย = ไม่ได้
code = "MOTOR"; 💥 คอมไพล์ไม่ผ่าน! (code คืออาร์เรย์ ย้ายบ้านไม่ได้)
ต้องใช้ strcpy(code, "MOTOR");

2️⃣ รับค่าด้วย scanf ไม่ต้องใส่ &
scanf("%s", name); — ชื่ออาร์เรย์ "เป็นตำแหน่ง" อยู่แล้ว (ข้อยกเว้นที่สัญญาไว้ตั้งแต่บทที่ 5!) และ %s หยุดรับที่ช่องว่างแรก

The \0 Inspector

🔎 เครื่องส่อง char array — เห็น \0 ด้วยตาตัวเอง

พิมพ์ข้อความ (อังกฤษ/ตัวเลข) แล้วดูว่าในหน่วยความจำหน้าตาเป็นอย่างไร: ตัวอักษรเรียงตามดัชนี และโบกี้สุดท้ายคือ \0 เสมอ — สังเกตความต่างระหว่าง strlen กับขนาดที่ต้องจองจริง

—
🧠

\0 สำคัญขนาดไหน?

printf("%s") และ strlen ทำงานด้วยวิธีเดียวกัน: วิ่งไปเรื่อย ๆ จนกว่าจะเจอ \0 — ถ้า \0 หาย (เช่น จองที่ไม่พอ) มันจะวิ่งเลยเขตไปอ่านหน่วยความจำข้างเคียงจนกว่าจะบังเอิญเจอศูนย์ ได้ข้อความขยะยาวพรืด — บั๊กสตริงเกือบทั้งหมดของภาษาซีคือเรื่อง \0 กับที่ไม่พอ

ตัวอย่างที่ 11.2 — count_char.c (วนจนกว่าจะเจอ \0)
char word[] = "BANANA";
int count = 0;
for (int i = 0; word[i] != '\0'; i++) {   // วนจนชน \0 พอดี
    if (word[i] == 'A') count++;
}
printf("ใน %s มีตัว A อยู่ %d ตัว\n", word, count);
The string.h Toolbox

🧰 กล่องเครื่องมือจาก <string.h>

สี่ฟังก์ชันหลักที่ใช้คู่กับสตริงทุกโปรแกรม — จุดที่พลาดบ่อยที่สุด: เปรียบเทียบข้อความต้องใช้ strcmp ไม่ใช่ ==

ฟังก์ชันหน้าที่ตัวอย่าง
strlen(s)นับความยาว (ไม่รวม \0)strlen("ENGINE") → 6
strcpy(a, b)คัดลอก b ทับลง astrcpy(code, "MOTOR");
strcat(a, b)ต่อ b ท้าย astrcat(name, "-01");
strcmp(a, b)เทียบเนื้อหา — เท่ากันได้ 0!strcmp(pass, "1234") == 0
isdigit(c)c เป็นตัวเลข? (ctype.h)isdigit(s[3])
isupper(c)c เป็นพิมพ์ใหญ่? (ctype.h)isupper(s[0])
ตัวอย่างที่ 11.3 — password.c (strcmp ของจริง)
char pass[20];
scanf("%s", pass);          // อาร์เรย์ไม่ใส่ &
if (strcmp(pass, "eng1234") == 0) {
    printf("เข้าสู่ระบบสำเร็จ\n");
} else {
    printf("รหัสผ่านไม่ถูกต้อง\n");
}
🚫

ทำไม == ใช้ไม่ได้?

if (pass == "1234") คอมไพล์ผ่านแต่ "เทียบตำแหน่งกล่อง" ไม่ใช่เนื้อหา — ได้เท็จแทบทุกครั้ง ต้องใช้ strcmp(a, b) == 0 (คืน 0 เมื่อเนื้อหาเท่ากัน — จุดที่สวนความรู้สึกที่สุดของบทนี้!)

📐

จัดรูปแบบรายงานด้วย printf

จัดคอลัมน์ให้ตรง: %-10s ข้อความชิดซ้ายกว้าง 10 ช่อง · %8.2f ตัวเลขชิดขวากว้าง 8 ทศนิยม 2 — printf("%-10s %8.2f\n", "Motor", 1250.5); คือหัวใจตารางรายงานทุกใบ

ตัวอย่างที่ 11.4 — strcat_len.c (ต่อข้อความ + วัดความยาว)
char code[20] = "PUMP";       // จองเผื่อไว้ 20 ช่อง
strcat(code, "-01");          // ต่อท้ายด้วย strcat (ห้ามใช้ +)
printf("รหัสเครื่อง: %s\n", code);
printf("ยาว %d ตัวอักษร\n", (int) strlen(code));
Putting It Together

🏷️ ใช้งานจริง: เครื่องตรวจรหัสสินค้า

คลังสินค้ากำหนดรหัสรูปแบบ XX-9999 (ตัวอักษรใหญ่ 2 ตัว, ขีด, ตัวเลข 4 หลัก) — พิมพ์รหัสแล้วดูโปรแกรมไล่ตรวจทีละกฎด้วยเครื่องมือของบทนี้ล้วน ๆ: strlen, ดัชนี, isupper, isdigit

🧪 ลองพิมพ์รหัส

strlen(code) == 7
isupper(code[0]) && isupper(code[1])
code[2] == '-'
isdigit(code[3]) ... isdigit(code[6])
—
ตัวอย่างที่ 11.5 — validate_code.c (โค้ดเบื้องหลังเครื่องนี้)
#include <string.h>
#include <ctype.h>

int is_valid(char code[]) {
    if (strlen(code) != 7) return 0;
    if (!isupper(code[0]) || !isupper(code[1])) return 0;
    if (code[2] != '-') return 0;
    for (int i = 3; i <= 6; i++)
        if (!isdigit(code[i])) return 0;
    return 1;
}

int main() {
    printf("AB-1234 → %d\n", is_valid("AB-1234"));
    printf("ab-1234 → %d\n", is_valid("ab-1234"));
    return 0;
}

สังเกตว่าใช้ความรู้ครบทุกบท: ฟังก์ชันคืน 1/0 (บท 9) + ดัชนีอาร์เรย์ (บท 10) + ลูปและเงื่อนไข (บท 6–7) — รหัสนักศึกษา เลขครุภัณฑ์ ทะเบียนรถ เลขพัสดุ ล้วนตรวจด้วยท่าเดียวกันนี้

Check Your Understanding

📝 แบบทดสอบท้ายบท (10 ข้อ)

จุดพลาดบ่อย: ลืมเผื่อช่อง \0 · ใช้ == แทน strcmp · ลืมว่า strcmp เท่ากันได้ 0

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

ความลับของบทนี้: สตริง = อาร์เรย์ char + ตัวปิดท้าย \0 — จึงต้องเผื่อช่อง +1 และใช้ฟังก์ชันเฉพาะ ห้ามใช้ = หรือ ==

1

สตริงคืออาร์เรย์ของ char + \0

สตริงคือชุดอักขระจบด้วยตัวปิดท้ายพิเศษ \0 (null) · เก็บ "ENGINE" 6 ตัว ต้องมีอย่างน้อย 7 ช่อง (เผื่อ \0)

char s[7] = "ENGINE";
// E N G I N E \0

2

strlen ไม่นับ \0

strlen("MOTOR-01") = 8 (นับเฉพาะตัวอักษรที่เห็น ไม่รวม \0)

3

เปลี่ยนค่า/เทียบสตริง ต้องใช้ฟังก์ชัน

ห้ามใช้ = กำหนดค่า หรือ == เทียบสตริง ต้องใช้ strcpy/strcmp

strcpy(name, "MIXER");        // เปลี่ยนค่า
if (strcmp(pass, "eng1234") == 0) // 0 = ตรงกัน

4

รับข้อความ + แก้ทีละตัว

รับด้วย scanf("%s", s) (ชื่ออาร์เรย์ไม่ต้องมี &) · แก้อักขระเดี่ยวผ่านดัชนี s[0] = 'X';

5

ctype: ตรวจชนิดอักขระ

isdigit(c), isupper(c) มาจาก <ctype.h> คืนค่าจริง/เท็จ · ใช้วนนับอักขระในสตริงได้

💡

จุดที่ข้อสอบชอบถาม

สองจุดตาย: ลืมเผื่อช่อง +1 สำหรับ \0 และเทียบสตริงด้วย == (ต้องใช้ strcmp) — โจทย์ตรวจรูปแบบรหัสสินค้าออกบ่อย
