อาร์เรย์ (Array)

ที่ผ่านมาตัวแปรหนึ่งตัวเก็บค่าได้ค่าเดียว — แต่งานจริงมีคะแนนนักศึกษา 40 คน พัสดุ 10,000 ชิ้น อุณหภูมิ 24 ชั่วโมง วันนี้พบ "อาร์เรย์": ตัวแปรเดียวที่เก็บข้อมูลทั้งขบวน และเป็นคู่หูที่ทำให้ลูปกับฟังก์ชันเปล่งพลังเต็มที่

Learning Objectives

วัตถุประสงค์การเรียนรู้

ประกาศอาร์เรย์ กำหนดค่าเริ่มต้น และเข้าถึงสมาชิกด้วยดัชนี (เริ่มที่ 0!) ได้
อธิบายข้อจำกัดของอาร์เรย์ภาษาซี: ขนาดตายตัว และอันตรายของการอ้างเกินขอบเขต
วนอ่านอาร์เรย์ด้วย for เพื่อหาผลรวม ค่าเฉลี่ย และค่าสูงสุดได้
เขียนฟังก์ชันที่รับอาร์เรย์เป็นพารามิเตอร์มาประมวลผลได้
อธิบายแนวคิดการจัดเรียงข้อมูล (Bubble Sort) และการสลับค่าด้วยตัวแปร temp ได้
Array Basics

🚃 อาร์เรย์ = ขบวนรถไฟของข้อมูลชนิดเดียวกัน

ประกาศด้วยวงเล็บเหลี่ยม int scores[4] = {80, 65, 90, 72}; — ทุกโบกี้เป็นชนิดเดียวกัน เรียงติดกันในหน่วยความจำ และกติกาที่สำคัญที่สุดของคาบนี้: ดัชนีเริ่มนับที่ 0 ไม่ใช่ 1!

ตัวอย่างที่ 10.1 — scores.c
int scores[4] = {80, 65, 90, 72};
printf("%d\n", scores[0]);   // ตัวแรก
printf("%d\n", scores[2]);   // ตัวที่สาม (ดัชนี 2!)
scores[1] = 70;              // เขียนทับโบกี้ที่สอง
printf("%d\n", scores[1]);
0️⃣

ดัชนีเริ่มที่ 0

อาร์เรย์ 4 ช่อง มีดัชนี 0, 1, 2, 3 — ตัวสุดท้ายคือ scores[3] ไม่ใช่ scores[4]! และภาษาซี ไม่มีดัชนีติดลบ แบบภาษาอื่น

💀

เกินขอบเขต = ไม่มีใครเตือน

อ้าง scores[10] ภาษาซี "ไม่ฟ้อง error" — มันไปอ่าน/เขียนหน่วยความจำข้างเคียงเฉย ๆ ได้ค่าขยะหรือโปรแกรมพังแบบสุ่ม (undefined behavior) — อันตรายที่สุดในคาบนี้ ผู้เขียนต้องคุมขอบเขตเอง

📏

ขนาดตายตัว

ประกาศ 4 ช่องได้ 4 ช่องตลอดชีพ ยืด-หดไม่ได้ — งานจริงจึงประกาศเผื่อ (เช่น 100 ช่อง) แล้วใช้ตัวแปร int n นับว่า "ใช้จริงกี่ช่อง"

ตัวอย่างที่ 10.2 — reverse.c (วนอ่านถอยหลังด้วย i--)
int t[5] = {10, 20, 30, 40, 50};
for (int i = 4; i >= 0; i--) {   // เริ่มท้าย (ขนาด-1) ถอยถึง 0
    printf("%d ", t[i]);
}
Interactive Playground

🧪 สนามทดลองอาร์เรย์ — เพิ่ม ลบ เข้าถึง เห็นผลทันที

อาร์เรย์ int arr[8] (ความจุ 8 ช่อง) กับตัวนับ n บอกจำนวนที่ใช้จริง — ท่ามาตรฐานของภาษาซี: เพิ่มท้ายด้วย arr[n] = ค่า; n++; ลบตัวท้ายด้วย n--; — สังเกตเลขดัชนีใต้กล่องทุกครั้งที่ข้อมูลเปลี่ยน

เริ่มต้น: int arr[8] = {12, 7, 25}; n = 3 — ลองกดปุ่มต่าง ๆ ได้เลย
Arrays + Loops + Functions

🤝 สามประสาน: อาร์เรย์ + ลูป + ฟังก์ชัน

ภาษาซี ไม่มี sum / max / len สำเร็จรูป — เราเขียนเองด้วยลูปจากคาบที่ 7 ซึ่งเป็นเรื่องดี: คุณจะเข้าใจไส้ในที่ภาษาอื่นซ่อนไว้ และนี่คือหน้าตาโปรแกรมระดับใช้งานจริง: ข้อมูลอยู่ในอาร์เรย์ → ฟังก์ชันรับอาร์เรย์ไปประมวลผล

ตัวอย่างที่ 10.3 — class_report.c
float average(int scores[], int n) {
    int sum = 0;
    for (int i = 0; i < n; i++)
        sum = sum + scores[i];
    return (float) sum / n;   // กัน int หาร int!
}

int main() {
    int points[5] = {72, 85, 61, 90, 77};
    for (int i = 0; i < 5; i++)
        printf("คะแนน %d → %s\n", points[i],
               points[i] >= 70 ? "ผ่าน" : "ไม่ผ่าน");
    printf("เฉลี่ย = %.2f\n", average(points, 5));
    return 0;
}

จำได้ไหมคาบที่ 7 เราเขียนลูปหาค่าสูงสุดเอง? — ในภาษาซีนั่นไม่ใช่แบบฝึกหัด แต่คือ "วิธีทำงานจริง" สังเกตด้วยว่าฟังก์ชันรับอาร์เรย์ต้องรับ "จำนวนสมาชิก n" ติดมาด้วยเสมอ เพราะตัวอาร์เรย์ไม่ได้พกขนาดของตัวเองไว้

ตัวอย่างที่ 10.4 — count.c (นับตามเงื่อนไข: ท่าประจำของงานสรุปข้อมูล)
int scores[6] = {72, 45, 88, 90, 51, 77};
int count = 0;
for (int i = 0; i < 6; i++) {
    if (scores[i] >= 70) count++;
}
printf("ผ่านเกณฑ์ %d คน จาก 6 คน\n", count);
Sorting

🫧 เครื่องจัดเรียงข้อมูล: Bubble Sort ทีละขั้น

อัลกอริทึมจัดเรียงที่เข้าใจง่ายที่สุด: เทียบคู่ที่ติดกัน ใครใหญ่กว่าสลับไปขวา วนซ้ำจนไม่มีอะไรให้สลับ — ตัวใหญ่จะค่อย ๆ "ลอย" ไปขวาเหมือนฟองอากาศ กดเทียบทีละคู่เอง หรือกดเล่นอัตโนมัติก็ได้

ข้อมูลตั้งต้น: {5, 2, 8, 3, 7, 1} — สีเหลือง = คู่ที่กำลังเทียบ, สีเขียว = เข้าที่แล้ว
ตัวอย่างที่ 10.5 — bubble_sort.c (หัวใจคือการสลับด้วย temp)
int arr[6] = {5, 2, 8, 3, 7, 1};
int n = 6;
for (int i = 0; i < n - 1; i++) {
    for (int j = 0; j < n - i - 1; j++) {
        if (arr[j] > arr[j + 1]) {
            int temp = arr[j];     // สลับค่า 3 จังหวะ
            arr[j] = arr[j + 1];
            arr[j + 1] = temp;
        }
    }
}
for (int i = 0; i < n; i++)
    printf("%d ", arr[i]);

🤔 ทำไมสลับต้องมี temp?

เขียน arr[j] = arr[j+1]; arr[j+1] = arr[j]; ตรง ๆ — ค่าแรกถูกทับหายไปก่อนจะย้าย! เหมือนเทน้ำสองแก้วสลับกัน ต้องมี "แก้วพักชั่วคราว" (temp) เสมอ

นี่คือแบบฝึก trace ลูปซ้อนที่ดีที่สุด และเป็นคำถามยอดนิยมทั้งในข้อสอบปลายภาคและห้องสัมภาษณ์งาน — ข้อสอบชอบให้ "ไล่ผลหลังจบรอบที่ 1" ลองกดเครื่องจัดเรียงดูให้เห็นกับตา

Check Your Understanding

📝 แบบทดสอบท้ายคาบ (10 ข้อ)

ท่องไว้ก่อนทำ: ดัชนีเริ่มที่ 0 · ช่องสุดท้ายคือ [ขนาด−1] · เกินขอบเขตไม่มีใครเตือน

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

กุญแจของคาบนี้คือ "ดัชนีเริ่มที่ 0" และตัวนับ n — เข้าใจแล้วเรื่องผลรวม/เฉลี่ย/Bubble Sort จะตามมาเอง

1

ประกาศอาร์เรย์ + ดัชนีเริ่มที่ 0

สมาชิกตัวแรกคือดัชนี 0 ตัวสุดท้ายคือ n-1

int items[3] = {10, 20, 30};
items[0]=10  items[1]=20  items[2]=30
// scores[5] ตัวสุดท้าย = scores[4]

2

อ้างเกินขอบเขต (out of bounds)

อ้าง scores[10] ทั้งที่มี 5 ช่อง ภาษาซีไม่เตือนแต่อ่านหน่วยความจำมั่ว → ผลลัพธ์เพี้ยน/โปรแกรมพัง

3

ตัวนับ n: จำนวนที่ใช้จริง

ประกาศเผื่อใหญ่ (เช่น int arr[100]) แต่ใช้จริงแค่บางช่อง — n บอกว่าใช้ไปกี่ตัว และใช้คุมลูป

arr[n] = x;   // เพิ่มต่อท้าย
n++;          // แล้วเลื่อนตัวนับ

4

วนหาผลรวม/เฉลี่ย + ส่ง n เข้าฟังก์ชัน

ภาษาซีไม่มีคำสั่งรวมอัตโนมัติ ต้องวนบวกเอง · ส่งฟังก์ชันต้องส่ง n ไปด้วยเพราะอาร์เรย์ไม่รู้ขนาดตัวเอง

float average(int s[], int n) { ... }

5

สลับค่า + Bubble Sort

การสลับต้องใช้ตัวแปรพัก temp · Bubble Sort เทียบคู่ติดกันแล้วสลับถ้าผิดลำดับ

int t = a[j]; a[j] = a[j+1]; a[j+1] = t;
// {5,2,8} คู่แรก (5,2) → สลับเป็น {2,5,8}

💡

จุดที่ข้อสอบชอบถาม

จำให้ขึ้นใจ: ดัชนีเริ่ม 0, ตัวสุดท้าย = n-1 · โจทย์ trace อาร์เรย์ (เช่น a[1]=a[3] แล้วบวกกัน) ออกบ่อย
