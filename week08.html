📋 ข้อมูลการสอบกลางภาค

🗺️

ขอบเขตเนื้อหา

บทที่ 1–7 ทั้งหมด: ประวัติคอมพิวเตอร์ · องค์ประกอบระบบและเลขฐาน · อัลกอริทึมและผังงาน · โครงสร้างภาษาซี ตัวแปร ชนิดข้อมูล printf · ตัวดำเนินการและ scanf · if–else if–else และ switch · for / while / do-while

🧾

รูปแบบข้อสอบ

3 ส่วนหลัก: ปรนัยวัดความเข้าใจ · ไล่ค่าตัวแปร (Trace) ทำนายผลลัพธ์ของโค้ด · เขียนโค้ด/ผังงานสั้น ๆ แก้โจทย์ — น้ำหนักอยู่ที่ Trace และเขียนโค้ดมากที่สุด

🎒

สิ่งที่ควรเตรียม

ปากกา ดินสอ ยางลบ บัตรนักศึกษา และ "กระดาษทดในหัว": ฝึกไล่ trace ด้วยมือจนคล่อง เพราะในห้องสอบไม่มีคอมไพเลอร์ให้รัน — สมองคุณคือคอมไพเลอร์

📌 วัน เวลา และห้องสอบ เป็นไปตามประกาศของมหาวิทยาลัย — ตรวจสอบใน LMS อีกครั้ง

7-Week Recap

🧠 แก่นของทั้ง 7 บท ในหน้าเดียว

การ์ดละบท สรุปเฉพาะที่ข้อสอบชอบถาม — คลิก "ทบทวนบทเต็ม →" เพื่อกลับไปเล่นตัวจำลองของบทนั้นอีกรอบ

🏛️

บทที่ 1: ประวัติคอมพิวเตอร์

วิวัฒนาการ 5 ยุค (หลอดสุญญากาศ → ทรานซิสเตอร์ → IC → ไมโครโพรเซสเซอร์ → AI) · กฎของมัวร์: ทรานซิสเตอร์เพิ่ม 2 เท่าทุก ~2 ปี · คอมพิวเตอร์ = เครื่อง "ประมวลผลตามคำสั่ง" ไม่ใช่เครื่องคิดเอง

ทบทวนบทเต็ม →

🖥️

บทที่ 2: องค์ประกอบระบบคอมพิวเตอร์

ครบ 4 ส่วน: ฮาร์ดแวร์ ซอฟต์แวร์ บุคลากร ข้อมูล · CPU = CU + ALU + Register · RAM ลบเลือน / ROM ถาวร · เลขฐานสอง: 1 Byte = 8 bit เก็บได้ 0–255 — ฝึกแปลงฐานให้คล่อง!

ทบทวนบทเต็ม →

🧭

บทที่ 3: อัลกอริทึมและผังงาน

คุณสมบัติ 5 ข้อ (เน้น Definiteness ชัดเจน + Finiteness ต้องจบ) · สัญลักษณ์: วงรีเริ่ม/จบ, ด้านขนาน I/O, สี่เหลี่ยมประมวลผล, ข้าวหลามตัดตัดสินใจ · ฝึก Trace ตามผังงานพร้อมตารางค่าตัวแปร

ทบทวนบทเต็ม →

🐍

บทที่ 4: เริ่มต้นเขียนโปรแกรมแรก

โครงสร้าง: #include <stdio.h> → int main() → { } จบทุกคำสั่งด้วย ; · ตัวแปรต้องประกาศชนิดก่อนใช้: int float double char · printf + ตัวกำหนดรูปแบบ %d %f %c %s และ %.2f · = คือกำหนดค่า ไม่ใช่เท่ากับ · ชื่อห้ามขึ้นต้นตัวเลข/เว้นวรรค/ภาษาไทย

ทบทวนบทเต็ม →

🧮

บทที่ 5: ตัวดำเนินการและนิพจน์

กับดักใหญ่: int หาร int ตัดเศษทิ้ง (5/9 = 0!) แก้ด้วย 5.0/9.0 · % เอาเศษ (n % 2 == 0 = เลขคู่) · ลำดับ: ( ) → * / % → + − · scanf ต้องมี & หน้าตัวแปรเสมอ · เปรียบเทียบให้ค่า 1/0 เชื่อมด้วย && || ! · == เปรียบเทียบ, = กำหนดค่า

ทบทวนบทเต็ม →

🔶

บทที่ 6: เลือกทำ if–else if–else และ switch

เงื่อนไขใน ( ) บล็อกใน { } — ห้ามใส่ ; ท้าย if! · ไล่บนลงล่าง เจอจริงตัวแรกทำแล้วหยุด → เรียงเงื่อนไขจากสูงลงต่ำ · else คือกรณีที่เหลือ ไม่มีเงื่อนไข · switch ใช้กับค่าตายตัว int/char ทุก case ต้องมี break ไม่งั้นไหลทะลุ

ทบทวนบทเต็ม →

🔁

บทที่ 7: วนซ้ำ for / while

for (เริ่มต้น; เงื่อนไข; ปรับค่า) — อ่านหัวลูปให้ครบสามช่อง · i++ คือ i = i + 1 · ท่าตัวสะสม total = total + i · while ตรวจก่อนทำ / do-while ทำก่อนตรวจ (อย่างน้อย 1 รอบ) · ลูปต้องขยับเข้าจุดจบ ไม่งั้นวนไม่รู้จบ · ลูปซ้อน: รอบรวม = นอก × ใน

ทบทวนบทเต็ม →

🔗

เส้นเรื่องที่ข้อสอบชอบ

ทุกอย่างต่อกันเป็นเรื่องเดียว: ผังงาน (บท 3) → โค้ด (บท 4) · นิพจน์ bool (บท 5) → เงื่อนไข if (บท 6) → เงื่อนไข while (บท 7) · ข้าวหลามตัด = if · ข้อสอบบูรณาการชอบให้ "แปลงผังงานเป็นโค้ด" และ "ไล่โค้ดที่มีทั้งลูปและ if"

Readiness Checklist

✅ เช็คลิสต์ความพร้อม 12 ข้อ

ติ๊กเฉพาะข้อที่ "ทำได้จริงโดยไม่เปิดดูเฉลย" — รายการจะถูกจำไว้ในเครื่อง กลับมาติ๊กเพิ่มได้เรื่อย ๆ จนกว่าจะครบก่อนวันสอบ

พร้อมสอบ 0 จาก 12 ข้อ
Mock Exam

📝 ข้อสอบจำลอง 10 ข้อ (คละบทที่ 1–7)

โจทย์ชุดใหม่ทั้งหมด ไม่ซ้ำกับแบบทดสอบท้ายบท — ทำเหมือนสอบจริง: ไม่เปิดสมุด ไม่ใช้คอมไพเลอร์ และจับเวลา 15 นาที

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

สอบกลางภาคครอบคลุมสัปดาห์ที่ 1–5 (บทที่ 1–7) นี่คือแผนที่ทบทวนแบบรวบยอด — กดกลับไปเล่นตัวจำลองของแต่ละบทได้

1

บทที่ 1–2 · ประวัติ + ฮาร์ดแวร์ + เลขฐาน

จำยุค–อุปกรณ์, บุคคลสำคัญ, RAM/ROM, และฝึกแปลงเลขฐาน

1101₂ = 8+4+0+1 = 13₁₀

2

บทที่ 3 · อัลกอริทึมและผังงาน

สัญลักษณ์ผังงาน (ข้าวหลามตัด=ตัดสินใจ, ด้านขนาน=รับ/แสดงผล), คุณสมบัติอัลกอริทึมที่ดี (ต้องมีจุดจบ), และการ trace

3

บทที่ 4 · โครงสร้างโปรแกรม + ชนิดข้อมูล

main(), #include, ชนิด int/float/char, ตัวกำหนดรูปแบบ · ตัวที่ทำให้คอมไพล์ไม่ผ่านแน่นอน = ลืม ; / ชื่อตัวแปรผิดกฎ

4

บทที่ 5 · ตัวดำเนินการ + scanf

กับดัก int division (7/2=3, 7.0/2=3.5) และ scanf ต้องมี & — ลืม & โปรแกรมพังตอนรัน

5

บทที่ 6–7 · เงื่อนไข + ลูป

if–else if (ทำบล็อกเดียว, ลำดับสำคัญ), switch+break, การนับรอบ for/while

int count = 0;
for (int i=1; i<=6; i++)
  if (i % 2 == 0) count++;
// count = 3 (i=2,4,6)

💡

จุดที่ข้อสอบชอบถาม

ข้อสอบกลางภาคชอบ "ไล่โค้ด (trace)" — ฝึกเดินทีละบรรทัดด้วยมือให้คล่อง โดยเฉพาะลูปกับเงื่อนไขผสมกัน

Exam Strategy

🎯 เทคนิคทำข้อสอบให้ได้คะแนนเต็มศักยภาพ

⏱️

กวาดง่ายก่อน

รอบแรกทำเฉพาะข้อที่มั่นใจให้หมด รอบสองค่อยกลับมาเก็บข้อยาก — อย่าจมกับข้อเดียวเกิน 3 นาที ทุกข้อคะแนนเท่ากัน

📝

Trace บนกระดาษเสมอ

ข้อไล่โค้ด ห้ามไล่ในหัว! เขียนตารางค่าตัวแปรแบบที่ฝึกกับตัวจำลองทุกบท — ช้ากว่า 30 วินาที แต่ถูกเกือบ 100%

🔍

ระวังกับดักคลาสสิก

สามจุดที่คนพลาดมากที่สุด: int หาร int ตัดเศษ (5/9 = 0) · scanf ลืม & · switch ลืม break ไหลทะลุ — เจอโจทย์แนวนี้ให้ช้าลงหนึ่งจังหวะ

✍️

ข้อเขียนโค้ด: โครงก่อนสวย

เขียนโครงให้ครบก่อน (รับค่า → แปลงชนิด → คำนวณ/เงื่อนไข → แสดงผล) แล้วค่อยเก็บรายละเอียด — โครงถูกได้คะแนนส่วนใหญ่แม้พลาดจุดเล็ก

😴

คืนก่อนสอบ

นอนให้พอสำคัญกว่าโต้รุ่ง — สมองล้าไล่ trace พลาดง่ายมาก ทบทวนรอบสุดท้ายแค่การ์ดสรุป 7 ใบข้างบน แล้วปิดหนังสือ

🧘

ในห้องสอบ

อ่านโจทย์ให้จบก่อนเขียน — ครึ่งหนึ่งของคะแนนที่หายไปมาจาก "ตอบไม่ตรงคำถาม" ไม่ใช่ "ไม่รู้" เช็คอีกรอบว่าโจทย์ถามหน่วยอะไร ปัดทศนิยมไหม

After the Exam

🚀 หลังสอบ: ครึ่งหลังที่สนุกกว่าเดิม

🏁

สิ่งที่คุณมีแล้วตอนนี้

สามโครงสร้างพื้นฐานครบมือ: ลำดับ เงื่อนไข วนซ้ำ — ในทางทฤษฎี คุณเขียน "ทุกโปรแกรมในโลก" ได้แล้ว ที่เหลือของคอร์สคือเครื่องมือที่ทำให้เขียนได้ "เร็วและเป็นระเบียบ" ขึ้น

🔮

ครึ่งหลังของคอร์ส

สัปดาห์ที่ 7 เริ่มด้วย "ฟังก์ชัน" — แบ่งโปรแกรมใหญ่เป็นชิ้นเล็กที่ใช้ซ้ำได้ ตามด้วยลิสต์ สตริง ไฟล์ และปิดท้ายด้วยโปรเจกต์จริงของแต่ละสาขา
