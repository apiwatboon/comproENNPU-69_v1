Algorithm

🧠 อัลกอริทึมคืออะไร?

อัลกอริทึม (Algorithm) คือ ลำดับขั้นตอนการแก้ปัญหาที่ชัดเจน ทำตามได้ และจบในเวลาจำกัด — ชื่อนี้มาจาก al-Khwarizmi นักคณิตศาสตร์ชาวเปอร์เซียในศตวรรษที่ 9 จริง ๆ แล้วคุณใช้อัลกอริทึมทุกวัน: สูตรทำอาหาร เส้นทางไปมหาวิทยาลัย วิธีซักผ้า ล้วนเป็นอัลกอริทึมทั้งสิ้น

🎯

1. ชัดเจนไม่กำกวม

"ใส่น้ำพอประมาณ" ❌ — "ใส่น้ำ 300 มล." ✅ คอมพิวเตอร์เดาใจไม่เป็น ทุกขั้นต้องระบุได้แน่นอน (Definiteness)

📥

2. มีข้อมูลเข้า–ออกชัดเจน

ระบุได้ว่ารับอะไรเข้า (Input ≥ 0 ตัว) และให้ผลลัพธ์อะไรออกมา (Output ≥ 1 ตัว)

🏁

3. ต้องจบ (Finiteness)

ทำงานเสร็จในจำนวนขั้นที่จำกัด — อัลกอริทึมที่วนไม่รู้จบคืออัลกอริทึมที่ผิด

🪜

4. เป็นลำดับขั้นตอน

ขั้นตอนเรียงต่อกันชัดเจน รู้เสมอว่าทำขั้นไหนก่อน–หลัง และขั้นถัดไปคืออะไร

⚙️

5. ปฏิบัติได้จริง

แต่ละขั้นต้องทำได้ด้วยทรัพยากรที่มี (Effectiveness) ไม่ใช่ "ขั้นที่ 3: เดาเลขที่ถูกต้อง"

🛣️

ปัญหาเดียว หลายอัลกอริทึม

ไปเชียงใหม่ได้ทั้งเครื่องบิน รถทัวร์ รถไฟ — ถูกทุกทางแต่ "ประสิทธิภาพ" ต่างกัน นี่คือหัวใจของวิทยาการคอมพิวเตอร์

Pseudocode

📝 รหัสเทียม (Pseudocode) — เขียนความคิดก่อนเขียนโค้ด

Pseudocode คือการเขียนอัลกอริทึมด้วยภาษาคนผสมโครงสร้างแบบโปรแกรม — ไม่มีไวยากรณ์ตายตัว ขอแค่อ่านแล้วเข้าใจตรงกัน ลองดูตัวอย่าง "คำนวณเกรดเฉลี่ย":

🗣️ แบบบรรยาย (ไม่ดี)

"ก็เอาคะแนนทุกวิชามารวมกันแล้วก็หารด้วยจำนวนวิชา ถ้าได้เยอะก็เกรดดี อะไรประมาณนั้น..."

⚠️ กำกวม: "เยอะ" คือเท่าไร? คะแนนมาจากไหน? แสดงผลยังไง?

✅ แบบ Pseudocode (ดี)

START
  READ จำนวนวิชา n
  SET sum = 0
  FOR i = 1 TO n
    READ คะแนนวิชาที่ i
    sum = sum + คะแนน
  END FOR
  SET average = sum / n
  PRINT average
END
Flowchart Symbols

🔷 สัญลักษณ์ผังงานมาตรฐาน 6 แบบ

ผังงาน (Flowchart) คืออัลกอริทึมฉบับ "วาดเป็นภาพ" — ใช้สัญลักษณ์สากล (ANSI/ISO) ที่วิศวกรทั่วโลกอ่านเข้าใจตรงกัน

Start / End

Terminator — เริ่ม/จบ

วงรีหรือทรงแคปซูล ทุกผังงานมีจุดเริ่ม 1 จุดและจุดจบอย่างน้อย 1 จุด

Input / Output

ข้อมูลเข้า/ออก

สี่เหลี่ยมด้านขนาน — รับข้อมูล (READ) หรือแสดงผล (PRINT)

Process

การประมวลผล

สี่เหลี่ยมผืนผ้า — การคำนวณหรือกำหนดค่า เช่น sum = a + b

?

การตัดสินใจ

สี่เหลี่ยมข้าวหลามตัด — ตรวจเงื่อนไข แตกทางออกเป็น "จริง / เท็จ"

→ ↓

ลูกศรทิศทาง

Flow line — บอกลำดับการไหลของงาน ปกติไหลจากบนลงล่าง ซ้ายไปขวา

A

จุดเชื่อมต่อ

วงกลม — เชื่อมผังงานที่ยาวเกินหน้ากระดาษ ให้ตามไปต่อที่จุดอักษรเดียวกัน

Interactive Simulator

🧭 ทดลองเดินผังงานทีละขั้น: หาค่ามากที่สุดของ 2 จำนวน

นี่คือทักษะ "Trace" — ไล่ตามผังงานทีละขั้นพร้อมจดค่าตัวแปร ลองเปลี่ยนค่า a, b แล้วกด "ก้าวถัดไป ▶" ดูเส้นทางของข้อมูล สังเกตว่าเมื่อ a, b เปลี่ยน เส้นทางก็เปลี่ยน!

START
รับค่า a, b
a > b ?
จริง ↓   |   เท็จ ↘
max = a
max = b
แสดงค่า max
END

🎛️ แผงควบคุม

ขั้นที่abmax
— กดก้าวถัดไปเพื่อเริ่ม —
พร้อมเริ่มเดินผังงาน — ลองตั้งค่า a ให้มากกว่า b แล้วดูว่าเส้นทางเปลี่ยนไหม
Worked Example

📐 ตัวอย่างครบวงจร: จากโจทย์ → อัลกอริทึม → ผังงาน

โจทย์: คำนวณดัชนีมวลกาย (BMI) แล้วแจ้งว่า "น้ำหนักเกิน" หรือ "ปกติ" (เกณฑ์ BMI ≥ 25 คือเกิน)

1️⃣

วิเคราะห์ปัญหา

Input: น้ำหนัก (กก.), ส่วนสูง (ม.)
Process: BMI = น้ำหนัก ÷ ส่วนสูง² แล้วเทียบกับ 25
Output: ค่า BMI + ข้อความแจ้งผล

2️⃣

เขียน Pseudocode

START
 READ weight, height
 bmi = weight / height²
 IF bmi ≥ 25 THEN
  PRINT "น้ำหนักเกิน"
 ELSE
  PRINT "ปกติ"
 END IF
END

3️⃣

วาดผังงาน

วงรี START → ด้านขนาน "รับ weight, height" → สี่เหลี่ยม "bmi = w/h²" → ข้าวหลามตัด "bmi ≥ 25?" แตกสองทางไปด้านขนานแสดงข้อความ → วงรี END — ลองวาดเองในสมุดตอนนี้เลย!

Check Your Understanding

📝 แบบทดสอบท้ายบท (10 ข้อ)

ข้อ Trace ลองใช้กระดาษทดไล่ค่าตัวแปรตามที่ฝึกในตัวจำลอง

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

บทนี้เน้น "สัญลักษณ์ผังงาน" และการ "ไล่ค่า (trace)" — จำรูปสัญลักษณ์และฝึกเดินตามขั้นตอนทีละบรรทัด

1

อัลกอริทึมคืออะไร + คุณสมบัติที่ดี

อัลกอริทึม = ลำดับขั้นตอนแก้ปัญหาที่ชัดเจนและมี จุดสิ้นสุด · คุณสมบัติที่ดี: ขั้นตอนชัดเจนไม่กำกวม, ทำได้จริง, และต้องจบ (โปรแกรมวนไม่จบ = ผิดคุณสมบัติข้อนี้)

2

สัญลักษณ์ผังงานที่ต้องจำ

ข้าวหลามตัด ◇ = ตัดสินใจ/เงื่อนไข · สี่เหลี่ยมผืนผ้า ▭ = ประมวลผล/คำนวณ · สี่เหลี่ยมด้านขนาน ▱ = รับเข้า/แสดงผล · วงรี ⬭ = เริ่ม/จบ

3

โครงสร้างควบคุม 3 แบบ

ทุกอัลกอริทึมประกอบจาก 3 โครงสร้าง: ลำดับ (Sequence), การเลือก (Selection/เงื่อนไข), การวนซ้ำ (Repetition/ลูป)

4

การไล่ค่า (Trace)

เดินทีละบรรทัด อัปเดตค่าตัวแปรตามลำดับ เช่น:

x = 5
x = x + 3   // x = 8
x = x × 2   // x = 16

5

Pseudocode ต่างจากภาษาจริง

Pseudocode เขียนด้วยภาษาคนเพื่อสื่อความคิด ไม่ต้องสนใจไวยากรณ์และคอมไพล์ไม่ได้ ส่วนภาษาโปรแกรมจริงต้องตรงไวยากรณ์เป๊ะ

💡

จุดที่ข้อสอบชอบถาม

โจทย์ "นับจำนวนข้าวหลามตัด" = นับจำนวนการตัดสินใจ (เงื่อนไข if แต่ละจุด) และโจทย์ trace ค่า — คิดทีละบรรทัดอย่าข้าม
