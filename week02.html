Learning Objectives

วัตถุประสงค์การเรียนรู้

เมื่อจบบทเรียนนี้ นักศึกษาจะสามารถ...

ระบุองค์ประกอบทั้ง 4 ของระบบคอมพิวเตอร์ และอธิบายความสัมพันธ์ระหว่างกันได้
จำแนกฮาร์ดแวร์ตามหน้าที่ 5 หน่วย และอธิบายบทบาทของ CPU, RAM, ROM และหน่วยเก็บข้อมูลได้
แยกความแตกต่างระหว่างซอฟต์แวร์ระบบกับซอฟต์แวร์ประยุกต์ พร้อมยกตัวอย่างได้
แปลงเลขระหว่างฐานสอง ฐานสิบ และฐานสิบหกได้ และอธิบายหน่วย bit/byte ได้
เรียงลำดับขั้นตอนการพัฒนาโปรแกรมทั้ง 6 ขั้นได้ เพื่อเตรียมพร้อมเขียนโปรแกรมจริงในสัปดาห์ถัดไป
The Computer System

ระบบคอมพิวเตอร์ = 4 องค์ประกอบ ที่ขาดกันไม่ได้

"คอมพิวเตอร์" เครื่องเปล่า ๆ ยังทำอะไรไม่ได้ — ระบบที่ทำงานได้จริงต้องมีครบทั้ง 4 ส่วนนี้

🔩

ฮาร์ดแวร์ (Hardware)

ตัวเครื่องและอุปกรณ์ที่จับต้องได้ — CPU, RAM, จอ, คีย์บอร์ด ถ้าระบบคือร่างกาย ฮาร์ดแวร์คือ "อวัยวะ"

💿

ซอฟต์แวร์ (Software)

ชุดคำสั่งที่บอกฮาร์ดแวร์ว่าต้องทำอะไร — จับต้องไม่ได้แต่ขาดไม่ได้ เปรียบเหมือน "ความคิด" ของระบบ

👥

บุคลากร (Peopleware)

ผู้ใช้ ผู้ดูแลระบบ และโปรแกรมเมอร์ — บทบาทที่คุณกำลังจะก้าวเข้ามาในวิชานี้

📊

ข้อมูล (Data)

วัตถุดิบของการประมวลผล — ตัวเลข ข้อความ ภาพ เสียง ที่ถูกแปลงเป็น 0/1 ให้เครื่องเข้าใจ

Hardware

🔩 ฮาร์ดแวร์ — แบ่งตามหน้าที่ได้ 5 หน่วย

โครงสร้างนี้คือสถาปัตยกรรม von Neumann จากบทที่ 1 ตัวจริง — คอมพิวเตอร์ทุกเครื่องตั้งแต่ ENIAC ถึงมือถือของคุณ ยังใช้ผังเดียวกันนี้

⌨️ 🖱️ 🎤

หน่วยรับข้อมูล (Input Unit)

คีย์บอร์ด เมาส์ ทัชสกรีน ไมโครโฟน สแกนเนอร์ เซนเซอร์ในโรงงาน

🧠

หน่วยประมวลผลกลาง (CPU)

สมองของเครื่อง — ทำงานวนลูป "อ่านคำสั่ง → ตีความ → ปฏิบัติ" หลายพันล้านรอบต่อวินาที

CUหน่วยควบคุม สั่งการทุกส่วน
ALUคำนวณ + เปรียบเทียบตรรกะ
Registerที่พักข้อมูลเร็วที่สุด
Cacheความจำใกล้ CPU ลดการรอ
หน่วยความจำหลัก (Main Memory) RAM — พื้นที่ทำงานชั่วคราว (ไฟดับ = หาย) · ROM — เก็บเฟิร์มแวร์ถาวรสำหรับบูตเครื่อง
หน่วยเก็บข้อมูลสำรอง (Storage) HDD / SSD / Flash Drive — เก็บถาวรแม้ปิดเครื่อง ความจุมากแต่ช้ากว่า RAM
🖥️ 🖨️ 🔊

หน่วยแสดงผล (Output Unit)

จอภาพ เครื่องพิมพ์ ลำโพง รวมถึงสัญญาณควบคุมแขนกลและสายพานการผลิต

💡 ลำดับความเร็ว (และราคาต่อหน่วย): Register → Cache → RAM → SSD → HDD — ยิ่งใกล้ CPU ยิ่งเร็วยิ่งแพง นี่คือเหตุผลที่เครื่องมีหน่วยความจำหลายชั้น

Software

💿 ซอฟต์แวร์ — ลำดับชั้นจากผู้ใช้ถึงฮาร์ดแวร์

ซอฟต์แวร์แบ่งใหญ่ ๆ ได้ 2 ประเภท: ซอฟต์แวร์ระบบ (จัดการเครื่อง) และซอฟต์แวร์ประยุกต์ (ทำงานให้ผู้ใช้) — ทุกอย่างเรียงเป็นชั้นแบบนี้

👤 ผู้ใช้ (User)คลิก พิมพ์ สั่งงาน
📱 ซอฟต์แวร์ประยุกต์ (Application Software)Word, Excel, เกม, เบราว์เซอร์, แอปมือถือ — และโปรแกรมที่คุณกำลังจะเขียน
⚙️ ซอฟต์แวร์ระบบ (System Software)ระบบปฏิบัติการ: Windows, macOS, Linux, Android · ตัวแปลภาษา: Compiler / Interpreter
🔩 ฮาร์ดแวร์ (Hardware)CPU, RAM, จอ, ดิสก์ — เข้าใจแค่ 0 กับ 1
⚙️

ซอฟต์แวร์ระบบ

ทำงานเบื้องหลัง จัดการทรัพยากรเครื่อง: ระบบปฏิบัติการแบ่งเวลา CPU จัดหน่วยความจำ จัดไฟล์ · ตัวแปลภาษา (Compiler/Interpreter) แปลโค้ดที่เราเขียนเป็นภาษาเครื่อง — เครื่องมือสำคัญที่สุดของโปรแกรมเมอร์

📱

ซอฟต์แวร์ประยุกต์

ทำงานเฉพาะด้านให้ผู้ใช้: โปรแกรมสำนักงาน เกม โปรแกรมออกแบบ CAD ระบบจัดการคลังสินค้า (WMS) ระบบวางแผนการผลิต (ERP) — งานวิศวกรรมยุคใหม่ขับเคลื่อนด้วยซอฟต์แวร์ประเภทนี้ทั้งสิ้น

Number Systems

🔢 ภาษาของเครื่อง — ระบบเลขฐาน

วงจรไฟฟ้ารู้จักแค่ 2 สถานะ: ไฟติด (1) กับไฟดับ (0) — ทุกอย่างในคอมพิวเตอร์ ทั้งรูป เพลง เกม และโค้ด จึงถูกเก็บเป็นเลขฐานสอง (Binary) ทั้งหมด ลองเล่นเครื่องแปลงด้านล่าง: คลิกที่บิตเพื่อสลับ 0/1 หรือพิมพ์เลขฐานสิบก็ได้

💡

bit — หน่วยเล็กที่สุด

1 bit = 0 หรือ 1 หนึ่งหลัก · 8 bit = 1 Byte เก็บอักษรอังกฤษได้ 1 ตัว · 1 KB = 1,024 Byte · 1 MB ≈ ล้าน Byte · 1 GB ≈ พันล้าน Byte

🔠

ฐานสิบหก — ทางลัดของโปรแกรมเมอร์

เลขฐานสองยาวเกินไป จึงนิยมย่อทีละ 4 บิตเป็นฐานสิบหก (0–9, A–F) เช่น 11111111₂ = FF₁₆ — เจอบ่อยในรหัสสี เช่น #22D3EE ของธีมหน้านี้

🏭

ทำไมวิศวกรต้องรู้?

PLC ในโรงงาน, รหัสบาร์โค้ด/RFID ในคลังสินค้า, เซนเซอร์ดิจิทัล — สื่อสารกันด้วยบิตทั้งหมด อ่านสเปกอุปกรณ์เป็น ต้องอ่านเลขฐานเป็น

Program Development

🛠️ 6 ขั้นตอนการพัฒนาโปรแกรม

โปรแกรมที่ดีไม่ได้เริ่มจากการพิมพ์โค้ด — มันเริ่มจากการ "เข้าใจปัญหา" หกขั้นตอนนี้คือกระดูกสันหลังของทั้งวิชา และเราจะใช้มันทุกครั้งที่ทำโจทย์

🔍

1. วิเคราะห์ปัญหา Problem Analysis

ระบุให้ชัด: ข้อมูลเข้า (Input) คืออะไร · ผลลัพธ์ (Output) ที่ต้องการคืออะไร · มีเงื่อนไข/ข้อจำกัดอะไร เช่น "คำนวณเกรด" ต้องรู้ช่วงคะแนนของแต่ละเกรดก่อน

📐

2. ออกแบบอัลกอริทึม Algorithm Design

วางลำดับขั้นการแก้ปัญหาเป็นข้อ ๆ ด้วย Pseudocode หรือผังงาน (Flowchart) — สัปดาห์หน้าเราจะฝึกเรื่องนี้แบบเต็ม ๆ

⌨️

3. เขียนโปรแกรม Coding

แปลงอัลกอริทึมเป็นภาษาโปรแกรมจริง — ขั้นนี้จะง่ายมากถ้าสองขั้นแรกทำมาดี และจะทรมานมากถ้าข้ามสองขั้นแรกมา

🧪

4. ทดสอบและแก้ไข Testing & Debugging

รันด้วยข้อมูลหลายแบบ รวมถึงกรณีสุดขั้ว (เช่น ใส่ 0, ใส่ค่าติดลบ) — ข้อผิดพลาดเรียกว่า "บั๊ก (Bug)" ตามตำนานแมลงเม่าที่บินเข้าไปติดในรีเลย์ของ Harvard Mark II ปี 1947

📄

5. จัดทำเอกสาร Documentation

เขียนคู่มือใช้งานและคอมเมนต์อธิบายโค้ด — เพื่อเพื่อนร่วมทีม และเพื่อตัวคุณเองในอีก 6 เดือนข้างหน้า ที่จะลืมว่าเคยเขียนอะไรไว้

🔧

6. บำรุงรักษา Maintenance

ปรับปรุงตามความต้องการใหม่และแก้บั๊กที่เจอภายหลัง — ในชีวิตจริง ขั้นนี้กินเวลากว่า 60% ของอายุซอฟต์แวร์ทั้งหมด

Check Your Understanding

📝 แบบทดสอบท้ายบท (10 ข้อ)

บางข้อใช้เครื่องแปลงเลขฐานด้านบนช่วยตรวจคำตอบได้ — ลองเลย!

📘 อ่านก่อนสอบ · สรุปประเด็นสำคัญ

โฟกัส 3 กลุ่ม: องค์ประกอบฮาร์ดแวร์/ซอฟต์แวร์ · หน่วยความจำ · และการแปลงเลขฐาน (คำนวณได้แน่นอนในข้อสอบ)

1

องค์ประกอบระบบ + ภายใน CPU

ระบบคอมพิวเตอร์ที่ครบ = ฮาร์ดแวร์ + ซอฟต์แวร์ + บุคลากร + ข้อมูล

ภายใน CPU มี 2 หน่วยหลัก: ALU (คำนวณ+เปรียบเทียบตรรกะ) และ CU (ควบคุมสั่งงาน)

2

หน่วยความจำ: RAM / ROM / Cache / สำรอง

RAM = ชั่วคราว เขียนได้ ไฟดับหาย · ROM = อ่านอย่างเดียว ไฟดับไม่หาย · Cache = เล็กแต่เร็วมาก พักข้อมูลที่ CPU ใช้บ่อย · หน่วยเก็บสำรอง = ฮาร์ดดิสก์/SSD/Flash Drive (เก็บถาวร)

3

ซอฟต์แวร์ระบบ vs ประยุกต์

ซอฟต์แวร์ระบบ = ระบบปฏิบัติการ (Windows/Linux), ตัวแปลภาษา · ซอฟต์แวร์ประยุกต์ = โปรแกรมใช้งาน เช่น Word, เกม

4

เลขฐาน: bit / byte และการแปลง

1 ไบต์ = 8 บิต · แปลงฐานสอง→ฐานสิบ คือถ่วงน้ำหนักด้วยเลขยกกำลังของ 2:

1011₂ = (1×8)+(0×4)+(1×2)+(1×1) = 11₁₀
200₁₀ = 11001000₂  (128+64+8)

5

ขั้นตอนพัฒนาโปรแกรม

เริ่มจาก วิเคราะห์ปัญหา เป็นขั้นแรกเสมอ → ออกแบบ → เขียนโค้ด → ทดสอบ → บำรุงรักษา

💡

จุดที่ข้อสอบชอบถาม

โจทย์คำนวณแปลงเลขฐานมาแน่ — ฝึกแปลงทั้งสองทาง และจำความต่าง RAM (หายเมื่อไฟดับ) กับ ROM (ไม่หาย) ให้ขึ้นใจ

ภาพรวมที่ต้องเห็น

ระบบคอมพิวเตอร์ = ฮาร์ดแวร์ + ซอฟต์แวร์ + บุคลากร + ข้อมูล · ฮาร์ดแวร์แบ่ง 5 หน่วยตามผัง von Neumann · ซอฟต์แวร์เรียงชั้นจากผู้ใช้ → แอป → OS → เครื่อง · ทุกข้อมูลคือเลขฐานสอง

แนวคิดสำคัญที่ต้องจำ

CPU = CU + ALU + Register · RAM ชั่วคราว / ROM ถาวร / Storage เก็บยาว · 1 Byte = 8 bit · ลำดับพัฒนาโปรแกรม: วิเคราะห์ → ออกแบบ → เขียน → ทดสอบ → เอกสาร → บำรุงรักษา

📌 งานมอบหมายสัปดาห์ที่ 1 (คาบที่ 2)

  1. เปิดสเปกคอมพิวเตอร์หรือโทรศัพท์ของตัวเอง ระบุ: CPU รุ่นอะไร / RAM เท่าใด / หน่วยเก็บข้อมูลชนิดและความจุเท่าใด แล้วจัดหมวดว่าแต่ละชิ้นอยู่ "หน่วย" ใดในผัง 5 หน่วย
  2. แปลงเลขต่อไปนี้ด้วยมือ (แสดงวิธีทำ) แล้วตรวจคำตอบด้วยเครื่องแปลงในหน้านี้: ① 156₁₀ → ฐานสอง ② 10110101₂ → ฐานสิบ ③ 255₁₀ → ฐานสิบหก
  3. เลือกงาน 1 อย่างในสาขาของคุณ (เช่น จัดตารางการผลิต, คำนวณเส้นทางขนส่ง) แล้วเขียน "6 ขั้นตอนการพัฒนาโปรแกรม" สำหรับงานนั้นแบบย่อ ขั้นละ 1–2 ประโยค
  4. ทำแบบทดสอบท้ายบทให้ได้อย่างน้อย 8/10 แล้วแคปหน้าจอคะแนนส่งใน LMS
