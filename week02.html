วัตถุประสงค์การเรียนรู้

เมื่อจบบทเรียนนี้ นักศึกษาจะสามารถ...

ระบุองค์ประกอบทั้ง 4 ของระบบคอมพิวเตอร์ และอธิบายความสัมพันธ์ระหว่างกันได้
จำแนกฮาร์ดแวร์ตามหน้าที่ 5 หน่วย และอธิบายบทบาทของ CPU, RAM, ROM และหน่วยเก็บข้อมูลได้
แยกความแตกต่างระหว่างซอฟต์แวร์ระบบกับซอฟต์แวร์ประยุกต์ พร้อมยกตัวอย่างได้
แปลงเลขระหว่างฐานสอง ฐานสิบ และฐานสิบหกได้ และอธิบายหน่วย bit/byte ได้
เรียงลำดับขั้นตอนการพัฒนาโปรแกรมทั้ง 6 ขั้นได้ เพื่อเตรียมพร้อมเขียนโปรแกรมจริงในสัปดาห์ถัดไป
The Computer System

ระบบคอมพิวเตอร์ = 4 องค์ประกอบ ที่ขาดกันไม่ได้

"คอมพิวเตอร์" เครื่องเปล่า ๆ ยังทำอะไรไม่ได้ — ระบบที่ทำงานได้จริงต้องมีครบทั้ง 4 ส่วนนี้
แตะหรือคลิกที่แต่ละองค์ประกอบ เพื่อดูรายละเอียด

องค์ประกอบที่ 1 · Hardware

ฮาร์ดแวร์ ตัวเครื่องที่จับต้องได้

ตัวเครื่องและอุปกรณ์ที่ "จับต้องได้" ทั้งหมด — CPU, RAM, จอภาพ, คีย์บอร์ด, เมาส์, ฮาร์ดดิสก์ ถ้าระบบคอมพิวเตอร์เปรียบเป็นร่างกาย ฮาร์ดแวร์ก็คือ "อวัยวะ" ที่ลงมือทำงานจริง

  • แบ่งตามหน้าที่ได้ 5 หน่วย: รับเข้า → ประมวลผล → ความจำ → เก็บข้อมูล → แสดงผล
  • มองเห็น จับต้องได้ และสึกหรอตามการใช้งาน
  • ลำพังฮาร์ดแวร์ทำอะไรไม่ได้ — ต้องมีซอฟต์แวร์คอยสั่งงาน
องค์ประกอบที่ 2 · Software

ซอฟต์แวร์ ชุดคำสั่งที่สั่งให้เครื่องทำงาน

ชุดคำสั่งที่บอกฮาร์ดแวร์ว่า "ต้องทำอะไร ทำอย่างไร" จับต้องไม่ได้แต่ขาดไม่ได้ เปรียบเหมือน "ความคิด" ของระบบที่คอยขับเคลื่อนอวัยวะทุกส่วน

  • แบ่งเป็น ซอฟต์แวร์ระบบ (OS, ตัวแปลภาษา) และ ซอฟต์แวร์ประยุกต์ (Word, เกม, แอป)
  • สร้างขึ้นด้วยการ "เขียนโปรแกรม" — ทักษะหลักของวิชานี้
  • แก้ไข อัปเดต และทำซ้ำได้ง่ายกว่าการเปลี่ยนฮาร์ดแวร์
องค์ประกอบที่ 3 · Peopleware

บุคลากร คนที่ใช้และสร้างระบบ

ผู้คนที่เกี่ยวข้องกับระบบ — ผู้ใช้งาน (User), ผู้ดูแลระบบ (Admin) และโปรแกรมเมอร์ (Programmer) ที่สร้างซอฟต์แวร์ขึ้นมาให้เครื่องทำงานได้

  • เป็นผู้กำหนดว่าจะให้ระบบทำอะไร และนำผลไปใช้ประโยชน์อย่างไร
  • โปรแกรมเมอร์คือบทบาทที่คุณกำลังจะก้าวเข้ามาในวิชานี้
  • เครื่องเก่งแค่ไหนก็ไร้ค่า ถ้าไม่มีคนสั่งงานอย่างถูกต้อง
องค์ประกอบที่ 4 · Data

ข้อมูล วัตถุดิบของการประมวลผล

วัตถุดิบของการประมวลผล — ตัวเลข ข้อความ ภาพ เสียง ที่ถูกแปลงเป็นเลขฐานสอง (0/1) ให้เครื่องเข้าใจ แล้วประมวลผลออกมาเป็นผลลัพธ์ที่มีความหมาย

  • ข้อมูลดิบ (Data) ถูกประมวลผลกลายเป็น สารสนเทศ (Information) ที่นำไปใช้ได้
  • ทุกอย่างในเครื่องถูกเก็บเป็นบิต — เชื่อมโยงกับเรื่องเลขฐานด้านล่าง
  • ข้อมูลเข้าผิด ผลลัพธ์ก็ผิด (Garbage In, Garbage Out)
Hardware

ฮาร์ดแวร์ — แบ่งตามหน้าที่ได้ 5 หน่วย

โครงสร้างนี้คือสถาปัตยกรรม von Neumann จากบทที่ 1 ตัวจริง — คอมพิวเตอร์ทุกเครื่องตั้งแต่ ENIAC ถึงมือถือของคุณ ยังใช้ผังเดียวกันนี้

แตะแต่ละหน่วยในแผนผังเพื่ออ่านรายละเอียด

เส้นข้อมูล (Data Bus) — รับ-ส่งข้อมูลสองทาง เส้นควบคุม (Control) — CPU สั่งงานทุกหน่วย

หน่วยรับข้อมูล Input Unit

ประตูทางเข้าของข้อมูล — แปลงสิ่งที่มนุษย์หรือสิ่งแวดล้อมป้อนเข้ามา ให้กลายเป็นสัญญาณดิจิทัล (0/1) ที่เครื่องเข้าใจได้ เป็นจุดเริ่มต้นของวงจรการทำงาน

คีย์บอร์ดเมาส์ทัชสกรีนไมโครโฟนสแกนเนอร์เซนเซอร์ในโรงงาน

หน่วยประมวลผลกลาง CPU — Central Processing Unit

"สมอง" ของเครื่อง ทำงานวนลูป อ่านคำสั่ง → ตีความ → ปฏิบัติ หลายพันล้านรอบต่อวินาที และเป็นตัวสั่งงานหน่วยอื่นทั้งหมด — ภายในแบ่งเป็น 4 ส่วนย่อย (กดป้าย CU / ALU / Register บนการ์ดด้านบนได้)

CU Control Unit · หน่วยควบคุม

ตัวสั่งการของ CPU — อ่านคำสั่งจากหน่วยความจำ ตีความว่าให้ทำอะไร แล้วสั่งหน่วยอื่นทำงานตามจังหวะ เปรียบเหมือน "วาทยกร" ที่คุมทั้งวง

ALU Arithmetic Logic Unit · หน่วยคำนวณ/ตรรกะ

หน่วยที่ลงมือคำนวณจริง — บวก ลบ คูณ หาร และเปรียบเทียบทางตรรกะ (>, <, =, AND/OR/NOT) ทุกการคำนวณในเครื่องเกิดขึ้นที่นี่

Register รีจิสเตอร์

ที่พักข้อมูลขนาดเล็กแต่ เร็วที่สุด อยู่ติดกับ ALU คอยถือข้อมูลและผลลัพธ์ที่กำลังใช้งาน ณ ขณะนั้น (เร็วกว่า RAM หลายเท่า แต่จุได้นิดเดียว)

Cache แคช

หน่วยความจำความเร็วสูงคั่นระหว่าง CPU กับ RAM เก็บข้อมูลที่ใช้บ่อยไว้ใกล้ ๆ เพื่อลดเวลาที่ CPU ต้องรอข้อมูลจาก RAM

หน่วยความจำหลัก Main Memory

พื้นที่ทำงานที่ CPU หยิบใช้ได้โดยตรงและรวดเร็ว — แบ่งเป็น RAM และ ROM

RAM — ชั่วคราว เขียนได้ ไฟดับหาย (volatile)ROM — อ่านอย่างเดียว เก็บเฟิร์มแวร์บูตเครื่องเร็วกว่าหน่วยเก็บสำรองมาก

หน่วยเก็บข้อมูลสำรอง Secondary Storage

เก็บข้อมูลและโปรแกรมไว้ "ถาวร" แม้ปิดเครื่อง ความจุมากแต่ช้ากว่า RAM — เป็นที่อยู่ของไฟล์งานทุกชิ้นของเรา

HDD — จานแม่เหล็กSSD — ชิปแฟลช เร็วกว่า HDDFlash Driveไม่หายเมื่อไฟดับ

หน่วยแสดงผล Output Unit

นำผลลัพธ์จากการประมวลผลออกมาให้มนุษย์รับรู้ หรือส่งสัญญาณไปควบคุมอุปกรณ์อื่น — เป็นปลายทางของวงจรการทำงาน

จอภาพเครื่องพิมพ์ลำโพงสัญญาณคุมแขนกล/สายพานการผลิต

Software

ซอฟต์แวร์ — ลำดับชั้นจากผู้ใช้ถึงฮาร์ดแวร์

ซอฟต์แวร์แบ่งใหญ่ ๆ ได้ 2 ประเภท: ซอฟต์แวร์ระบบ (จัดการเครื่อง) และซอฟต์แวร์ประยุกต์ (ทำงานให้ผู้ใช้) — ทุกอย่างเรียงเป็นชั้นแบบนี้

ผู้ใช้ (User)คลิก พิมพ์ สั่งงาน
ซอฟต์แวร์ประยุกต์ (Application Software)Word, Excel, เกม, เบราว์เซอร์, แอปมือถือ — และโปรแกรมที่คุณกำลังจะเขียน
ซอฟต์แวร์ระบบ (System Software)3 กลุ่ม: ระบบปฏิบัติการ · ตัวแปลภาษา (Compiler/Interpreter) · โปรแกรมอรรถประโยชน์
ฮาร์ดแวร์ (Hardware)CPU, RAM, จอ, ดิสก์ — เข้าใจแค่ 0 กับ 1
ซอฟต์แวร์ระบบ System Software · ทำงานเบื้องหลัง จัดการเครื่อง

แบ่งย่อยได้ 3 กลุ่ม:

ระบบปฏิบัติการ (OS)Windows, macOS, Linux, Android — แบ่งเวลา CPU จัดหน่วยความจำ จัดไฟล์
ตัวแปลภาษา (Language Translator)Compiler · Interpreter · Assembler — แปลโค้ดที่เราเขียนเป็นภาษาเครื่อง เครื่องมือสำคัญของโปรแกรมเมอร์
โปรแกรมอรรถประโยชน์ (Utility)แอนตี้ไวรัส, บีบอัด/แตกไฟล์, สำรองข้อมูล, จัดการดิสก์
ซอฟต์แวร์ประยุกต์ Application Software · ทำงานเฉพาะด้านให้ผู้ใช้

โปรแกรมที่ทำงานเฉพาะด้านให้ผู้ใช้ — งานวิศวกรรมยุคใหม่ขับเคลื่อนด้วยซอฟต์แวร์กลุ่มนี้ และเป็นสิ่งที่คุณกำลังจะเขียน

เจาะลึก “ตัวแปลภาษา” 3 แบบ — Compiler · Interpreter · Assembler

ทั้งสามแปลโค้ด → ภาษาเครื่อง (0/1) เหมือนกัน แต่ต่างกันที่ “แปลจากภาษาอะไร และวิธีแปล”

Compiler คอมไพเลอร์
โค้ด .c→ Compiler→ โปรแกรม .exe→ รัน
  • แปล ทั้งโปรแกรมทีเดียว ได้ไฟล์ภาษาเครื่อง (.exe) ออกมา
  • รัน เร็ว เพราะแปลเสร็จไปก่อนแล้ว
  • แก้โค้ดต้อง compile ใหม่ ทุกครั้ง
  • เจอข้อผิดพลาดตั้งแต่ตอน compile (ก่อนรัน)
ตัวอย่างภาษา: C, C++, Go, Rust
Interpreter อินเทอร์พรีเตอร์
โค้ด .py→ Interpreterอ่าน+รันทีละบรรทัด→ ผลลัพธ์
  • แปล + รัน ทีละบรรทัด ขณะทำงาน (ไม่สร้างไฟล์ .exe)
  • รัน ช้ากว่า เพราะแปลใหม่ทุกครั้งที่รัน
  • แก้โค้ดแล้ว รันได้เลย ไม่ต้อง compile
  • เจอข้อผิดพลาดตอนรันมาถึงบรรทัดนั้น
ตัวอย่างภาษา: Python, JavaScript, PHP
Assembler แอสเซมเบลอร์
โค้ด .asm→ Assembler→ ภาษาเครื่อง
  • แปล ภาษาแอสเซมบลี → ภาษาเครื่อง (เกือบ 1 คำสั่งต่อ 1 คำสั่ง)
  • ได้โปรแกรม เล็กและเร็วมาก
  • ผูกกับ CPU แต่ละรุ่น (ย้ายเครื่องไม่ได้)
  • ใช้กับงานที่ต้องคุมฮาร์ดแวร์ละเอียด / ระบบฝังตัว
ตัวอย่าง: NASM, MASM, GAS

เคล็ดลับคือ “ค่าประจำหลัก” — แต่ละหลักทางซ้ายมีค่าเป็น 2 เท่า ของหลักทางขวา · หลักไหนเป็น 1 ก็ “นับ” ค่านั้น, เป็น 0 ก็ข้าม แล้วรวมกัน (ไม่ต้องท่องสูตรยกกำลัง)

ASCII — เก็บ “ตัวอักษร” เป็นเลขฐานสองได้อย่างไร

คอมพิวเตอร์เก็บได้แค่ตัวเลข 0/1 — เวลาจะเก็บ “ตัวอักษร” จึงใช้รหัส ASCII กำหนดเลขประจำตัวให้อักษรแต่ละตัว เช่น A = 65 = 01000001 · a = 97 · 0 (ศูนย์ที่เป็นตัวอักษร) = 48

หน่วยข้อมูล: bit · byte และทำไม “ขนาดไฟล์” กับ “ความเร็วเน็ต” หน่วยไม่เหมือนกัน

8 bit = 1 byte

bit (b ตัวเล็ก) = หนึ่งหลัก 0/1 · byte (B ตัวใหญ่) = 8 bit รวมกัน เก็บตัวอักษรอังกฤษได้ 1 ตัว — ระวัง! b เล็ก ≠ B ใหญ่

หน่วยฝั่ง bit ก็มี Kb, Mb, Gb เหมือนกัน — ต่างกันแค่ตัว b เล็ก = bit

6 ขั้นตอนการพัฒนาโปรแกรม

โปรแกรมที่ดีไม่ได้เริ่มจากการพิมพ์โค้ด — มันเริ่มจากการ "เข้าใจปัญหา" หกขั้นตอนนี้คือกระดูกสันหลังของทั้งวิชา และเราจะใช้มันทุกครั้งที่ทำโจทย์

1

วิเคราะห์ปัญหา Problem Analysis

ระบุให้ชัด: ข้อมูลเข้า (Input) คืออะไร · ผลลัพธ์ (Output) ที่ต้องการคืออะไร · มีเงื่อนไข/ข้อจำกัดอะไร เช่น "คำนวณเกรด" ต้องรู้ช่วงคะแนนของแต่ละเกรดก่อน

2

ออกแบบอัลกอริทึม Algorithm Design

วางลำดับขั้นการแก้ปัญหาเป็นข้อ ๆ ด้วย Pseudocode หรือผังงาน (Flowchart) — สัปดาห์หน้าเราจะฝึกเรื่องนี้แบบเต็ม ๆ

3

เขียนโปรแกรม Coding

แปลงอัลกอริทึมเป็นภาษาโปรแกรมจริง — ขั้นนี้จะง่ายมากถ้าสองขั้นแรกทำมาดี และจะทรมานมากถ้าข้ามสองขั้นแรกมา

4

ทดสอบและแก้ไข Testing & Debugging

รันด้วยข้อมูลหลายแบบ รวมถึงกรณีสุดขั้ว (เช่น ใส่ 0, ใส่ค่าติดลบ) — ข้อผิดพลาดเรียกว่า "บั๊ก (Bug)" ตามตำนานแมลงเม่าที่บินเข้าไปติดในรีเลย์ของ Harvard Mark II ปี 1947

5

จัดทำเอกสาร Documentation

เขียนคู่มือใช้งานและคอมเมนต์อธิบายโค้ด — เพื่อเพื่อนร่วมทีม และเพื่อตัวคุณเองในอีก 6 เดือนข้างหน้า ที่จะลืมว่าเคยเขียนอะไรไว้

6

บำรุงรักษา Maintenance

ปรับปรุงตามความต้องการใหม่และแก้บั๊กที่เจอภายหลัง — ในชีวิตจริง ขั้นนี้กินเวลากว่า 60% ของอายุซอฟต์แวร์ทั้งหมด

1

องค์ประกอบระบบ + ภายใน CPU

ระบบที่ครบ = ฮาร์ดแวร์ + ซอฟต์แวร์ + บุคลากร + ข้อมูล · ภายใน CPU: CU (สั่งงาน) · ALU (คำนวณ+เปรียบเทียบ) · Register (ที่พักเร็วสุด) · Cache

2

หน่วยความจำ + บัส

RAM ชั่วคราว ไฟดับหาย · ROM ถาวร · Cache เล็กแต่เร็ว คั่น CPU–RAM (มี hit/miss) · Storage (SSD/HDD) เก็บถาวร · ทุกการอ่าน/เขียน RAM ต้องส่ง address ก่อน

3

ซอฟต์แวร์ · ตัวแปลภาษา · ระดับภาษา

ระบบ (OS, ตัวแปลภาษา, utility) vs ประยุกต์ (Word/เกม) · ตัวแปลภาษา: Compiler (ทั้งโปรแกรม→.exe, C ใช้ตัวนี้) · Interpreter (ทีละบรรทัด, Python) · Assembler · ระดับภาษา: สูง (C) → แอสเซมบลี → ภาษาเครื่อง

4

เลขฐาน: แปลง · 0–F · การบวก

1 ไบต์ = 8 บิต · แปลงฐานสอง→ฐานสิบ คือ บวกค่าประจำหลักที่เป็น 1:

0111₂ → หลักที่เป็น 1: 4 + 2 + 1 = 7₁₀
ฐานสิบหก: 0-9 แล้ว A-F (10-15) · 4 บิต = 1 หลัก
การบวก: 1 + 1 = 10₂ (ได้ 0 ทด 1)

5

ASCII + หน่วยข้อมูล

ASCII = เลขประจำตัวอักษร (A=65=01000001) · ขนาดไฟล์วัดเป็น Byte (KB/MB) · ความเร็วเน็ตวัดเป็น bit/วินาที (Mbps) — เน็ต 100 Mbps ≈ 12.5 MB/s (หาร 8)
